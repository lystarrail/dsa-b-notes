一、本章考什么、怎么学

核心结论：递归题的关键不是“看见函数自己调自己”，而是看三个问题：有没有结束条件？问题规模有没有变小？变小后是不是同一个类型的问题？

1. 本章知识地图

递归三定律→数列求和→进制转换→调用栈→汉诺塔/迷宫
模块你要懂什么常见考法
递归概念把问题分解为规模更小的相同问题。判断是不是递归、找基本结束条件。
递归三定律基本结束条件、向结束条件演进、调用自身。代码填空、解释无限递归原因。
任意进制转换递归先处理高位，再接低位。写出 toStr 输出，解释 // 和 %。
调用栈每次递归调用都会压入一个栈帧，返回时弹出。递归深度、RecursionError、二分查找调用栈容量。
汉诺塔先搬 n-1 个，再搬最大盘，再搬 n-1 个。移动次数、递归代码、递推式。
迷宫探索递归尝试四个方向，需要标记已访问。基本结束条件、为什么需要“面包屑”。

2. 小白学习法

递归先找“最小问题”

写代码前先问：小到什么程度可以直接返回？比如列表只剩 1 个数、盘子只剩 1 个、到达迷宫出口。

再找“怎么变小”

每次调用必须离最小问题更近。列表变短，整数商变小，汉诺塔盘子数 n-1，迷宫走到新的格子。

最后看“返回怎么合并”

递归不只是向下调用，还要向上返回。比如 toStr(n//base) 返回高位，再拼上当前余数。

代码题要画调用链

递归代码看不懂时，把每一层调用写成一行，标出参数怎么变化、哪一层先返回。

二、什么是递归与递归三定律

1. 递归是什么

递归 Recursion：一种解决问题的方法。它把一个大问题分解为“规模更小的相同问题”，持续分解，直到问题小到可以直接解决；在程序中表现为函数调用自身。

2. 递归三定律

定律意思不满足会怎样
有基本结束条件最小规模问题能直接解决没有出口，会无限递归
向结束条件演进每次调用都让问题变小或状态更接近结束参数不变，也会无限递归
调用自身用同一个函数解决更小的同类问题不是真正的递归
判断技巧：看到递归代码，先不要急着逐行运行。先圈出 if 直接 return 的基本结束条件，再看递归调用里的参数有没有变小。

三、初识递归：数列求和

1. 把“求一串数的和”拆成更小问题

求 [1,3,5,7,9] 的和，可以理解成：

sum([1,3,5,7,9]) = 1 + sum([3,5,7,9])
sum([3,5,7,9]) = 3 + sum([5,7,9])
一直拆到列表只剩 1 个数，直接返回。

2. 代码解释

代码解释：递归数列求和
def listsum(numList):
    if len(numList) == 1:
        return numList[0]              # 基本结束条件：只剩一个数，直接返回
    else:
        return numList[0] + listsum(numList[1:])
        # 规模变小：从原列表变成去掉第一个元素的列表

print(listsum([1, 3, 5, 7, 9]))        # 25

例题1：给 listsum 找递归三定律

题目：说明 listsum 这个递归函数的基本结束条件、规模减小方式和自身调用分别是什么。

参考答案：基本结束条件是 len(numList)==1；规模减小方式是使用 numList[1:]，列表长度减少；自身调用是 listsum(numList[1:])。

四、递归应用：任意进制转换

1. 进制转换的递归思路

核心结论：把整数 n 转成 base 进制时，n % base 得到当前最低位，n // base 是剩下的高位问题。递归先处理高位，再拼最低位。

2. 代码解释

代码解释：递归版任意进制转换
def toStr(n, base):
    convertString = "0123456789ABCDEF"
    if n < base:
        return convertString[n]       # 基本结束条件：一位就能表示
    else:
        return toStr(n // base, base) + convertString[n % base]
        # 先递归处理高位，再拼接当前最低位

print(toStr(1453, 16))                # 5AD

五、递归调用的实现与深度限制

1. 递归为什么和栈有关

调用栈：每调用一次函数，系统都会把这次调用的“现场”暂时保存起来，形成一个栈帧。这里的“现场”可以理解成：这个函数算到哪一步了、参数是多少、等下一层递归返回后还要继续拼什么。函数返回时，最上面的栈帧先弹出，所以递归是后调用的先返回。
先把图读法说清楚：左边不是最终答案，而是“函数一层层调用下去，系统先把还没做完的函数保存起来”；右边才是“最小问题先返回，然后一层层把答案拼回去”。
执行到哪一层为什么还不能直接返回它在等什么返回后做什么
toStr(1453,16)1453 不是一位十六进制数等 toStr(90,16) 算出高位把自己的余数 13 转成 D 拼在后面
toStr(90,16)90 也不是一位十六进制数等 toStr(5,16) 算出高位把自己的余数 10 转成 A 拼在后面
toStr(5,16)5 已经小于 16不需要再递归直接返回 5
按时间顺序看：先调用 toStr(1453)，它没法立即完成，于是调用 toStr(90)；toStr(90) 也没法立即完成，于是调用 toStr(5)；toStr(5) 到达基本结束条件，先返回 5。然后上一层接着算出 5A，再上一层接着算出 5AD。

2. Python 的递归深度限制

RecursionError：直接原因是递归调用层数超过 Python 允许的最大递归深度，也就是系统调用栈装不下了。更根本的原因通常有三类：忘记写基本结束条件；状态没有向基本结束条件靠近，导致无限递归；问题规模太大或递归写法本身不够合适，使得递归层数过深。
代码解释：查看和设置递归深度
import sys
print(sys.getrecursionlimit())   # 常见默认值约为 1000
sys.setrecursionlimit(3000)      # 可以调整，但不是万能解法

例题2：递归二分查找需要多大的调用栈？

题目思想：给定 N 个相异元素构成的有序数列，用递归二分查找。递归过程中调用栈容量大约需要多少？

参考答案：数量级是 O(log N)。因为每次二分查找都把问题规模缩小约一半。若把初始调用也算作一层，最大栈帧数通常可理解为 ⌈log₂N⌉ + 1 左右；做选择题时要看选项是否把初始调用算入。

六、递归应用：汉诺塔

1. 问题规则

有三根柱子，若干盘子从小到大叠在起始柱上。目标是把所有盘子移到目标柱。规则是：一次只能移动一个盘子；大盘子不能压在小盘子上。

递归思路：要搬 n 个盘子，先把上面 n-1 个盘子搬到中间柱，再把最大盘搬到目标柱，最后把 n-1 个盘子从中间柱搬到目标柱。

2. 代码解释

代码解释：汉诺塔
def moveTower(height, fromPole, toPole, withPole):
    if height >= 1:
        moveTower(height - 1, fromPole, withPole, toPole)
        moveDisk(fromPole, toPole)
        moveTower(height - 1, withPole, toPole, fromPole)

def moveDisk(fp, tp):
    print("moving disk from", fp, "to", tp)

moveTower(3, "A", "C", "B")

七、递归应用：探索迷宫

1. 迷宫如何建模

迷宫可以看成一个二维矩阵。每个格子有行列坐标，格子内容可能是墙壁 +、通道、起点 S、已经走过的位置等。

2. 递归探索思路

当前位置是墙？失败→走过？失败→出口？成功→尝试四个方向

3. 伪代码理解

代码解释：迷宫搜索的递归骨架
def searchFrom(maze, row, col):
    if maze[row][col] == '+':          # 墙壁
        return False
    if maze[row][col] == '.':          # 面包屑：走过了
        return False
    if maze.isExit(row, col):          # 到达边缘出口
        return True

    maze[row][col] = '.'               # 撒面包屑，标记已访问

    found = (searchFrom(maze, row-1, col) or
             searchFrom(maze, row+1, col) or
             searchFrom(maze, row, col-1) or
             searchFrom(maze, row, col+1))
    return found

例题3：探索迷宫的基本结束条件

题目：迷宫递归搜索中，哪些情况可以直接返回，不再继续递归？

参考答案：碰到墙壁返回失败；碰到已经访问过的“面包屑”返回失败；到达边缘出口返回成功；四个方向都失败时返回失败。
