二、队列 Queue：FIFO 与 Python 实现

1. 什么是队列

队列 Queue：一种有次序的数据集合。新数据项总从队尾 rear 加入，已有数据项总从队首 front 移除。它的规则是 先进先出 FIFO，也就是先到先服务。
队列只有一个入口和一个出口，不允许从中间插队或移除。

2. Queue ADT 的操作

操作作用是否修改队列
Queue()创建空队列创建新对象
enqueue(item)把 item 加入队尾会
dequeue()从队首移除并返回数据项会
isEmpty()判断是否为空队列不会
size()返回队列长度不会

3. 用 Python list 实现 Queue

课程实现中，把 list 的 0 号位置作为队尾，把 list 末端作为队首。这样 dequeue() 可以直接 pop()，通常是 O(1)；但 enqueue() 要 insert(0,item)，需要整体移动元素，通常是 O(n)。

代码解释：Queue 的 list 实现
class Queue:
    def __init__(self):
        self.items = []

    def isEmpty(self):
        return self.items == []

    def enqueue(self, item):
        self.items.insert(0, item)   # 在 list 首端插入：队尾 rear

    def dequeue(self):
        return self.items.pop()      # 从 list 末端弹出：队首 front

    def size(self):
        return len(self.items)

q = Queue()
q.enqueue(4)
q.enqueue('dog')
q.enqueue(True)
print(q.dequeue())   # 4，最早进入的先出来
易错点：不要把“队首/队尾”和 list 的“首端/末端”混在一起。ADT 只规定队列行为；具体实现可以把任意一端设成 front/rear，但复杂度会跟着改变。

例题1：队列操作模拟

题目思想：依次执行 enqueue(4)、enqueue('dog')、enqueue(True)、dequeue()，返回谁？队列剩什么？

参考答案：返回 4，因为 4 最早进入队列；剩余队列按课程 list 表示为 [True, 'dog']，其中 'dog' 更靠近队首。

三、队列应用：热土豆问题

1. 问题怎么理解

热土豆问题类似“击鼓传花”：一群人围成圈传土豆，传过固定次数后，当前拿到土豆的人出局。继续传，直到只剩一个人。

为什么用队列：队首表示当前拿着土豆的人。传一次，就是队首出队，然后马上从队尾入队；传满 num 次后，队首出队且不再入队。

2. 代码解释

代码解释：热土豆 hotPotato
def hotPotato(namelist, num):
    simqueue = Queue()
    for name in namelist:
        simqueue.enqueue(name)

    while simqueue.size() > 1:
        for i in range(num):
            # 传一次土豆：队首的人出队，马上去队尾
            simqueue.enqueue(simqueue.dequeue())

        # 传了 num 次后，队首这个人出局
        simqueue.dequeue()

    return simqueue.dequeue()

print(hotPotato(['Bill','David','Susan','Jane','Kent','Brad'], 7))

例题2：热土豆过程怎么模拟

题目思想：为什么 simqueue.enqueue(simqueue.dequeue()) 表示“一次传递”？

参考答案：dequeue() 取出当前队首，也就是当前拿土豆的人；随后 enqueue() 把这个人放回队尾，相当于他把土豆传给下一人后排到圈尾等待。传满固定次数后，再执行一次 dequeue()，此时队首的人被淘汰。

四、双端队列 Deque 与回文词判定

1. 什么是双端队列

双端队列 Deque：一种有次序的数据集，两端都可以加入数据项，也都可以移除数据项。它集成了栈和队列的一些能力，但本身没有固定的 LIFO 或 FIFO 规则。
操作作用
Deque()创建空双端队列
addFront(item)从队首加入
addRear(item)从队尾加入
removeFront()从队首移除并返回
removeRear()从队尾移除并返回
isEmpty() / size()判断空、返回长度

2. Python list 实现 Deque

代码解释：Deque 的一种实现
class Deque:
    def __init__(self):
        self.items = []

    def isEmpty(self):
        return self.items == []

    def addFront(self, item):
        self.items.append(item)       # list 末端作为 front

    def addRear(self, item):
        self.items.insert(0, item)    # list 首端作为 rear

    def removeFront(self):
        return self.items.pop()

    def removeRear(self):
        return self.items.pop(0)

    def size(self):
        return len(self.items)
易错点：Deque “能两端操作”不等于自动 FIFO 或 LIFO。用它模拟栈/队列时，要由使用者自己统一操作规则。

3. 回文词判定

回文词正读和反读都一样，比如 radar、madam。用双端队列可以把字符从两端取出来比较：首尾相同就继续，不同就失败。

代码解释：回文词判定
def palchecker(aString):
    chardeque = Deque()
    for ch in aString:
        chardeque.addRear(ch)

    stillEqual = True
    while chardeque.size() > 1 and stillEqual:
        first = chardeque.removeFront()
        last = chardeque.removeRear()
        if first != last:
            stillEqual = False

    return stillEqual

print(palchecker('radar'))   # True
print(palchecker('python'))  # False

例题3：回文词为什么适合用 Deque？

题目思想：判断 radar 是否是回文词，为什么用双端队列方便？

参考答案：回文词需要不断比较最前和最后两个字符。Deque 支持从两端移除，正好能每轮取出首尾字符并比较；radar 中第一轮 r=r，第二轮 a=a，最后只剩 d，所以是回文词。

五、无序表 List 与链表思想

1. 什么是无序表

无序表 Unordered List：一种按相对位置存放的数据集。它叫“无序”，不是说没有顺序，而是说数据项不按大小排序，只按存放位置形成第 1 个、第 2 个、最后一个这样的关系。

比如分数集合 54, 26, 93, 17, 77, 31 用无序表表示为 [54, 26, 93, 17, 77, 31]。

2. List ADT 的常用操作

操作作用
List()创建空列表
add(item)添加一个数据项，假设原先不存在
remove(item)移除指定数据项
search(item)查找数据项是否存在
isEmpty() / size()判空、求长度
append/index/insert/pop表尾添加、返回位置、指定位置插入、删除末尾/指定位置

3. 为什么可以用链表实现无序表

无序表要求保持数据项的相对位置，但不要求它们在内存里连续摆放。链表通过“节点 + 链接”保持前后关系：每个节点保存数据和指向下一个节点的引用。

例题4：哪个属于存储结构？

题目思想：下列哪个概念属于存储结构？A. 线性表 B. 链表 C. 栈 D. 队列

参考答案：B. 链表。线性表、栈、队列主要描述逻辑关系或操作规则；链表描述的是数据在内存中如何通过节点和链接来存放，属于链式存储结构。

六、Node 与 UnorderedList 的链表实现

1. Node 节点是什么

链表节点 Node：至少包含两部分：数据项本身 data，以及指向下一个节点的引用 next。当 next is None，表示后面没有节点了。
代码解释：Node 类
class Node:
    def __init__(self, initdata):
        self.data = initdata
        self.next = None

    def getData(self):
        return self.data

    def getNext(self):
        return self.next

    def setData(self, newdata):
        self.data = newdata

    def setNext(self, newnext):
        self.next = newnext

2. UnorderedList 为什么要有 head

链表不能像 Python list 那样通过下标直接跳到某个位置。要访问所有节点，必须从第一个节点开始沿着 next 一直走，所以无序表对象需要一个 head 保存第一个节点。

代码解释：UnorderedList 基础
class UnorderedList:
    def __init__(self):
        self.head = None       # 空表没有第一个节点

    def isEmpty(self):
        return self.head == None

3. add：为什么加到表头最快

无序表不要求大小顺序，所以新节点放哪里都可以。链表最容易插入的位置是表头，因为只需要修改新节点和 head，不需要遍历到表尾。

代码解释：add 到链表表头
def add(self, item):
    temp = Node(item)
    temp.setNext(self.head)    # 第一步：新节点指向原来的第一个节点
    self.head = temp           # 第二步：head 改为指向新节点
顺序很重要：必须先让 temp.next 指向原 head，再移动 head。如果先 self.head = temp，原链表入口就丢了。

4. size 和 search：遍历整条链

size：数节点
def size(self):
    current = self.head
    count = 0
    while current != None:
        count += 1
        current = current.getNext()
    return count
search：逐个找
def search(self, item):
    current = self.head
    found = False
    while current != None and not found:
        if current.getData() == item:
            found = True
        else:
            current = current.getNext()
    return found

5. remove：为什么需要 previous

删除某个节点时，不能只知道当前节点 current，还要知道它前面的节点 previous。因为真正的删除动作是：让 previous.next 跳过 current，指向 current.next。

代码解释：remove(item)
def remove(self, item):
    current = self.head
    previous = None
    found = False

    while not found:
        if current.getData() == item:
            found = True
        else:
            previous = current
            current = current.getNext()

    if previous == None:
        # 删除的是第一个节点
        self.head = current.getNext()
    else:
        # 删除的是中间或尾部节点
        previous.setNext(current.getNext())

例题5：单链表中把 s 插到 p 后面

题目思想：在单链表中，要将 s 所指节点插入到 p 所指节点之后，应该怎样改指针？

参考答案：s.next = p.next; p.next = s。必须先让 s 接上 p 原来的后继，再让 p 指向 s；否则会丢掉原来的后半段链表。

七、OrderedList 有序表实现

1. 什么是有序表

有序表 OrderedList：数据项按照某种可比较性质决定位置。以整数为例，越小的数据越靠近表头，越大的越靠后。

例如 17 → 26 → 31 → 54 → 77 → 93 就是一个按升序排列的有序表。

2. 有序表和无序表哪些方法一样

isEmpty、size、remove 与节点是否有序关系不大，所以实现思路基本和无序表一致。真正变化比较大的是 search 和 add。

3. search：可以提前停止

无序表找不到一个元素时，通常要搜完整条链。有序表可以更聪明：如果当前节点的数据已经大于目标值，那么后面的数据只会更大，目标不可能再出现，可以提前返回 False。

代码解释：OrderedList.search
def search(self, item):
    current = self.head
    found = False
    stop = False

    while current != None and not found and not stop:
        if current.getData() == item:
            found = True
        else:
            if current.getData() > item:
                stop = True        # 后面只会更大，不用继续找
            else:
                current = current.getNext()
    return found

4. add：找到第一个比 item 大的节点

有序表插入必须保持整体顺序。比如在 17, 26, 54, 77, 93 中插入 31，要找到第一个比 31 大的节点 54，然后把 31 插到 54 前面。

代码解释：OrderedList.add
def add(self, item):
    current = self.head
    previous = None
    stop = False

    while current != None and not stop:
        if current.getData() > item:
            stop = True
        else:
            previous = current
            current = current.getNext()

    temp = Node(item)
    if previous == None:
        temp.setNext(self.head)
        self.head = temp
    else:
        temp.setNext(current)
        previous.setNext(temp)

例题6：有序单链表插入的平均复杂度

题目思想：在一个具有 n 个结点的有序单链表中插入一个新结点并仍保持有序，平均时间复杂度是多少？

参考答案：O(n)。因为单链表不能随机跳到插入位置，必须从 head 开始沿 next 向后找第一个比新元素大的节点。平均可能要看大约一半节点，数量级仍是 O(n)。

八、链表复杂度与试题应用

1. 链表复杂度看什么

判断原则：如果一个操作只改 head 或少数几个引用，通常是 O(1)；如果必须从 head 沿 next 遍历，通常是 O(n)。
操作复杂度原因
isEmpty()O(1)只检查 head is None。
无序表 add(item)O(1)直接插入表头，不用找位置。
size()O(n)必须遍历到表尾才能数清节点。
search(item)O(n)最坏要找完整条链。
remove(item)O(n)先要找到 item，同时维护 previous。
有序表 add(item)O(n)要找到正确插入位置。

2. 链表和 Python 内置 list 的区别

不要混淆：这里讲的链表是“节点 + next”的链式存储；Python 内置 list 是顺序存储并做了优化。两者名字都像“列表”，但底层完全不同，所以同名操作复杂度可能不一样。

3. 链表反转例题：指针怎么改

例题7：单链表反转填空

题目思想：已知单链表用 self.head 直接指向第一个数据节点。现在要把链表原地反转。做法可以理解成：从原链表头部一个个摘节点，再用“头插法”插到新链表表头。请补全下面代码中的 ①②③。

def reverse(self):
    p = self.head              # p 指向当前要搬运的节点
    self.head = None           # 先把新链表看成空表

    while p is not None:
        temp = p               # 暂存当前节点
        p = ________           # ① 先保存原链表剩余部分入口
        temp.next = ________   # ② 当前节点接到新链表表头前
        ________ = temp        # ③ 更新新表头

参考答案：

def reverse(self):
    p = self.head
    self.head = None

    while p is not None:
        temp = p
        p = p.next             # ① 先让 p 指向下一个节点，防止后半段链表丢失
        temp.next = self.head  # ② 当前节点接到新链表表头前
        self.head = temp       # ③ 当前节点成为新的表头

理解：这段代码的本质就是反复做“头插”。p 像搬运工，指向原链表中当前要处理的节点；p = p.next 必须先做，因为拆掉当前节点之前要先记住剩余链表入口；temp.next = self.head 把当前节点接到新链表前面；self.head = temp 再更新表头。核心是：改指针前，先保存后续节点，避免把后半段链表弄丢。

4. 循环单链表例题：只有 tail 怎么找头

例题8：只带表尾指针的循环单链表表头插入

题目思想：循环单链表中，tail 指向最后一个节点，最后一个节点的 next 指向第一个节点，也就是 tail.next 是表头。现在要在表头插入新节点 nd，请补全下面代码。

def pushFront(self, data):
    nd = Node(data)
    if self.tail is None:
        self.tail = nd
        nd.next = ________          # ① 空表时自己指向自己
    else:
        nd.next = ________          # ② 新节点先指向原表头
        ________ = nd               # ③ 尾节点再指向新表头

参考答案：

def pushFront(self, data):
    nd = Node(data)
    if self.tail is None:
        self.tail = nd
        nd.next = nd
    else:
        nd.next = self.tail.next
        self.tail.next = nd

理解：空表时只有一个节点，所以 nd.next = nd 才能形成循环；非空时，必须先令 nd.next = self.tail.next，让新节点接上原表头，再令 self.tail.next = nd，让尾节点指向新表头。顺序反了就可能丢失原表头入口。
