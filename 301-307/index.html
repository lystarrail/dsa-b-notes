二、线性结构：有序、两端、前驱后继

1. 什么是线性结构

线性结构：一种有序数据项集合。除第一个数据项没有前驱、最后一个没有后继外，每个数据项都有唯一前驱和唯一后继。
图示理解：线性结构像排队、串珠、车厢；数据项只按一个方向排成序列。

2. 栈、队列、双端队列、列表都属于线性结构

它们的共同点是：数据项之间只存在先后次序关系。区别在于：从哪一端加入、从哪一端移除、是否允许从两端操作。

结构主要限制直觉
栈 Stack同一端加入和移除盘子堆：最后放上去的最先拿走
队列 Queue一端加入，另一端移除排队：先来先服务
双端队列 Deque两端都可加入/移除两头都能操作的队列
列表 List更一般的线性表，可在指定位置插入、删除、查找一串有顺序的数据

例题1：哪个是存储结构？

题目思想：下列哪个概念属于存储结构？A. 线性表 B. 链表 C. 栈 D. 队列

参考答案：B. 链表。链表强调结点和指针如何存放，是存储结构；线性表、栈、队列主要描述逻辑关系或操作规则。

三、栈 Stack：LIFO 与 ADT

1. 什么是栈

栈 Stack：一种有次序的数据项集合，数据项的加入和移除都只发生在同一端。这一端叫栈顶 top，另一端叫栈底 base。

2. 栈的核心特性：后进先出 LIFO

距离栈底越近的数据项，留在栈中的时间越长；最新加入的数据项在栈顶，会被最先移除。这就是 后进先出，英文叫 Last In First Out，缩写 LIFO。

生活例子：浏览器“后退”按钮先回到最近访问的网页；Word 的 Undo 先撤销最近一步操作；盘子堆最上面的盘子最先被拿走。

3. ADT Stack 的操作

操作作用会不会修改栈
Stack()创建一个空栈创建新对象
push(item)把 item 加入栈顶会
pop()移除并返回栈顶数据项会
peek()返回栈顶数据项但不移除不会
isEmpty()判断是否为空栈不会
size()返回栈中元素个数不会

4. 手动模拟栈操作

例子：

s=Stack() → []
push(4) → [4]
push('dog') → [4,'dog']
peek() 返回 'dog'，但栈仍是 [4,'dog']
pop() 返回 'dog'，栈变成 [4]

例题2：进栈/出栈后栈顶是谁？

题目思想：空栈 S，待进栈序列为 a,b,c,d,e,f，依次执行：进栈、进栈、出栈、进栈、进栈、出栈。操作完成后，栈顶元素是谁？

过程：push a → [a] push b → [a,b] pop → [a] push c → [a,c] push d → [a,c,d] pop → [a,c]

参考答案：栈顶是 c。

四、Python 实现 Stack 与复杂度

1. 用 class 实现 ADT Stack

Python 可以用类来实现用户自定义类型。Stack 是一个数据集，可以用 Python 原生 list 当内部容器。

代码解释：list 末端作为栈顶
class Stack:
    def __init__(self):
        self.items = []              # 内部用 list 保存数据；使用者不需要直接操作它

    def isEmpty(self):
        return self.items == []

    def push(self, item):
        self.items.append(item)      # 把 list 末端当作栈顶：append 到末端

    def pop(self):
        return self.items.pop()      # 从 list 末端弹出

    def peek(self):
        return self.items[len(self.items) - 1]

    def size(self):
        return len(self.items)

s = Stack()
s.push(4)
s.push('dog')
print(s.peek())      # dog
print(s.pop())       # dog
print(s.size())      # 1
代码理解：self.items 是内部实现；push/pop/peek 是对外接口。使用者调用接口，不需要关心里面到底是不是 list。

2. 为什么选择 list 末端作为栈顶

易错点：不要直接说“Stack 的 pop 一定是 O(1)”。严格说，ADT Stack 只规定行为，不规定实现。用 list 末端实现时 pop() 通常是 O(1)；如果用 list 首端实现，pop(0) 通常是 O(n)。

例题3：为什么“ADT Stack 的 pop 一定是 O(1)”不严谨？

题目思想：栈的接口是固定的，为什么还不能直接断言某个接口一定是什么复杂度？

参考答案：因为复杂度属于具体实现，不属于 ADT 接口本身。若用 list 末端作栈顶，append/pop 通常 O(1)；若用 list 首端作栈顶，insert(0)/pop(0) 会移动元素，通常 O(n)。

五、栈应用：括号匹配

1. 为什么括号匹配适合用栈

核心结论：最新打开的左括号，应该最先匹配后面遇到的右括号。这种“最近的先处理”正好符合栈的 LIFO。

2. 简单圆括号匹配算法

从左到右扫描→遇到 '(' 入栈→遇到 ')' 弹栈→中途不能空弹→最后栈要空
代码解释：简单括号匹配
def parChecker(symbolString):
    s = Stack()
    balanced = True
    index = 0

    while index < len(symbolString) and balanced:
        symbol = symbolString[index]
        if symbol == '(':
            s.push(symbol)              # 左括号：先存起来，等待后面匹配
        else:                           # 这里只考虑 ')' 的简单版本
            if s.isEmpty():
                balanced = False        # 没有左括号可配，失败
            else:
                s.pop()                 # 用一个右括号匹配掉最近的左括号
        index += 1

    return balanced and s.isEmpty()     # 扫完后不能剩下未匹配的左括号

3. 通用括号匹配：()、[]、{}

如果有三类括号，遇到右括号时不仅要看栈是否为空，还要看栈顶左括号和当前右括号是否属于同一类。

代码解释：通用括号匹配
def matches(open_ch, close_ch):
    opens = '([{'
    closes = ')]}'
    return opens.index(open_ch) == closes.index(close_ch)


def parChecker(symbolString):
    s = Stack()
    balanced = True
    index = 0

    while index < len(symbolString) and balanced:
        symbol = symbolString[index]
        if symbol in '([{':
            s.push(symbol)
        else:
            if s.isEmpty():
                balanced = False
            else:
                top = s.pop()
                if not matches(top, symbol):
                    balanced = False
        index += 1

    return balanced and s.isEmpty()

例题4：判别左右括号是否配对，最好用什么结构？

题目思想：设计一个判别表达式中左、右括号是否配对出现的算法，采用哪种数据结构最佳？

参考答案：栈。左括号入栈，右括号出现时弹出并匹配栈顶左括号；最后栈空且没有中途失败，则括号配对成功。

六、栈应用：十进制转换为二进制/任意进制

1. 为什么进制转换要用栈

十进制整数转二进制时，采用“除以 2 求余数”。每次余数是当前最低位，所以得到的顺序是：低位 → 高位。但输出二进制时要从高位 → 低位，所以需要栈来反转顺序。

2. 十进制转二进制代码

代码解释：除 2 取余 + 栈反转
def divideBy2(decNumber):
    remstack = Stack()

    while decNumber > 0:
        rem = decNumber % 2       # 当前最低位
        remstack.push(rem)        # 先存起来
        decNumber = decNumber // 2

    binString = ''
    while not remstack.isEmpty():
        binString += str(remstack.pop())

    return binString

print(divideBy2(233))     # 11101001

3. 扩展到 2~16 进制

把“除以 2”改成“除以 base”，余数可能是 0~15。为了表示十六进制中的 10~15，需要使用 A B C D E F。

代码解释：转换到十六以下任意进制
def baseConverter(decNumber, base):
    digits = '0123456789ABCDEF'
    remstack = Stack()

    while decNumber > 0:
        rem = decNumber % base
        remstack.push(rem)
        decNumber = decNumber // base

    newString = ''
    while not remstack.isEmpty():
        newString += digits[remstack.pop()]

    return newString

print(baseConverter(233, 2))    # 11101001
print(baseConverter(233, 8))    # 351
print(baseConverter(233, 16))   # E9
抓住一句话：除基取余得到的顺序和输出顺序相反，栈就是用来完成这个反转。

七、中缀、前缀、后缀表达式

1. 中缀表达式为什么麻烦

我们平常写 A+B*C，操作符在操作数中间，所以叫中缀。中缀表达式符合人类习惯，但计算机处理时要考虑优先级和括号。

表达式类型操作符位置例子
中缀操作符在操作数中间A+B
前缀操作符在操作数前面+AB
后缀操作符在操作数后面AB+

2. 为什么后缀表达式适合计算机

在后缀表达式里，操作符出现在需要计算的两个操作数之后。这样从左到右扫描时，只要遇到操作符，就能从栈顶取出最近的两个操作数进行计算，不必再考虑括号和复杂优先级。

例子：(A+B)*C 的后缀形式是 A B + C *。先看到 A B +，知道先算 A+B，再与 C 相乘。

八、中缀转后缀算法

1. 全括号思路：先理解转换本质

如果中缀表达式已经全括号化，比如 (A+(B*C))，每个右括号都对应一个完整子表达式。把操作符移动到对应右括号的位置，就得到后缀表达式。

例子：A+B*C 全括号化为 (A+(B*C))，转成后缀就是 A B C * +。

2. 通用算法：操作符栈 opstack

核心结论：操作数直接输出；操作符不能马上确定输出位置时，先放入操作符栈。遇到优先级低/相等的新操作符、右括号或扫描结束时，再把合适的操作符弹出。
操作数→直接进输出列表|左括号→入操作符栈|右括号→弹到左括号
代码解释：中缀转后缀
def infixToPostfix(infixexpr):
    prec = {}
    prec['*'] = 3
    prec['/'] = 3
    prec['+'] = 2
    prec['-'] = 2
    prec['('] = 1

    opStack = Stack()
    postfixList = []
    tokenList = infixexpr.split()

    for token in tokenList:
        if token in 'ABCDEFGHIJKLMNOPQRSTUVWXYZ' or token in '0123456789':
            postfixList.append(token)       # 操作数：直接输出
        elif token == '(':
            opStack.push(token)             # 左括号：先标记起来
        elif token == ')':
            topToken = opStack.pop()
            while topToken != '(':
                postfixList.append(topToken)
                topToken = opStack.pop()
        else:                               # 操作符 + - * /
            while (not opStack.isEmpty()) and (prec[opStack.peek()] >= prec[token]):
                postfixList.append(opStack.pop())
            opStack.push(token)

    while not opStack.isEmpty():
        postfixList.append(opStack.pop())

    return ' '.join(postfixList)

print(infixToPostfix('A + B * C'))          # A B C * +
print(infixToPostfix('( A + B ) * C'))      # A B + C *

3. 做题时怎么手算

扫描到怎么处理原因
操作数直接写入后缀表达式操作数顺序通常不改变。
左括号压入操作符栈括号内部运算优先。
右括号弹出操作符直到左括号括号内子表达式结束。
普通操作符和栈顶操作符比优先级，必要时弹出高优先级/同级左结合的操作符应先输出。

例题5：中缀转后缀过程中操作符栈状态

题目思想：利用栈将表达式 3*2^(4+2*2-6*3)-5 转换为后缀表达式过程中，当扫描到 6 时，运算符栈是什么？

简化过程：扫描到 3*2^(4+2*2- 后，栈中保留外层 *、^、左括号 ( 和当前括号内的 -。接着扫描到 6 时，6 是操作数，直接输出，不改变操作符栈。

参考答案：*^(-。

九、后缀表达式求值

1. 为什么后缀求值适合用栈

核心结论：从左到右扫描后缀表达式；操作数先压入栈，遇到操作符时，弹出最近的两个操作数计算，再把结果压回去。

2. 后缀求值代码

代码解释：后缀表达式求值
def doMath(op, op1, op2):
    if op == '*':
        return op1 * op2
    elif op == '/':
        return op1 / op2
    elif op == '+':
        return op1 + op2
    else:
        return op1 - op2


def postfixEval(postfixExpr):
    operandStack = Stack()
    tokenList = postfixExpr.split()

    for token in tokenList:
        if token in '0123456789':
            operandStack.push(int(token))
        else:
            operand2 = operandStack.pop()    # 右操作数：先弹出
            operand1 = operandStack.pop()    # 左操作数：后弹出
            result = doMath(token, operand1, operand2)
            operandStack.push(result)

    return operandStack.pop()

print(postfixEval('4 5 6 * +'))    # 34

3. 为什么 operand2 先弹出

关键易错点：对于 7 3 -，扫描到 - 时，先弹出 3，它是右操作数；后弹出 7，它是左操作数。所以计算是 7-3，不是 3-7。

例题6：后缀表达式对应的中缀表达式

题目思想：给定后缀表达式 ab+-c*d-，其对应的中缀表达式是哪一个？

参考答案：-(a+b)*c-d。这里 ab+ 先形成 (a+b)，紧接着的 - 按题目答案理解为一元负号，得到 -(a+b)，再与 c 相乘，最后减 d。

例题7：扫描到第二个 + 前，栈里有什么？

题目思想：使用栈计算后缀表达式 1 2 3 + 4 * 5 + 3 + −，当扫描到第二个 + 但还未对该 + 运算时，栈的内容是什么？要求从栈底到栈顶写。

过程：1 → [1] 2 → [1,2] 3 → [1,2,3] + → [1,5] 4 → [1,5,4] * → [1,20] 5 → [1,20,5]。此时扫描到第二个 +，还没计算。

参考答案：1, 20, 5。
