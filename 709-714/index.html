一、本章考什么、怎么学

本章主线：701-708 学了图的表示、BFS、DFS 入门；709-714 继续学习更通用的 DFS，以及基于图的几个典型算法：拓扑排序、强连通分支、Dijkstra、Prim。
DFS深度优先森林
拓扑排序DAG 的依赖顺序
SCC互相可达的最大顶点群
Dijkstra正权最短路径
Prim最小生成树

1. 这章算法在解决什么问题

算法输入是什么输出/目标是什么一句话理解
通用 DFS任意图DFS 树/森林、发现时间、结束时间一条路尽量走到底，走不动再回退。
拓扑排序有向无圈图 DAG满足依赖关系的线性序列谁必须先做，就排在前面。
强连通分支有向图若干互相可达的顶点集合把“内部来回都通”的点分成一组。
Dijkstra带正权图 + 起点起点到各顶点的最短距离每次先确定当前距离最短的点。
Prim带权连通无向图连接所有顶点且总权重最小的树从一棵小树开始，每次接最便宜的安全边。

2. 学习顺序

通用 DFS→结束时间→拓扑排序→强连通分支→Dijkstra / Prim
易混提醒：Dijkstra 和 Prim 都常用优先队列，但 Dijkstra 关心“从起点到某点的最短距离 dist”，Prim 关心“把树外某点接进当前树的最小边权”。

二、通用 DFS：发现时间与结束时间

1. 通用 DFS 和骑士周游 DFS 有什么区别

骑士周游中的 DFS

目标是找一条包含所有格子的路径，像一棵没有分支的“退化树”。走不通就回溯，换下一条路。

通用 DFS

目标是在图中尽量深入搜索，连接尽量多的顶点。图可能不连通，所以可能生成多棵树，叫 DFS 森林。

通用 DFS 的核心：把每个顶点从白色（未发现）变成灰色（已发现，正在探索），最后变成黑色（探索完成）。同时记录前驱、发现时间和结束时间。

2. 三个状态与三个属性

内容含义代码中常见名字
颜色 colorwhite 未发现；gray 已发现但还没完成；black 已完成。color
前驱 predecessor从哪个顶点走到当前顶点，用来形成 DFS 树/森林。pred
发现时间 discovery第一次访问到这个顶点的时间。disc
结束时间 finish当前顶点的所有邻接点都处理完的时间。fin

3. 发现时间和结束时间的括号性质

括号性质：如果 v 是 u 的 DFS 子孙，那么 u 会比 v 更早被发现，也会比 v 更晚结束。也就是 u.discovery < v.discovery < v.finish < u.finish。这像一对外层括号包住内层括号。

例题1：判断 DFS 时间关系

题目：如果在 DFS 树中，B 是 A 的子节点，那么 A 和 B 的发现/结束时间有什么关系？

答案：A 先被发现，B 后被发现；B 先结束，A 后结束。写成不等式是 A.d < B.d < B.f < A.f。

三、DFS 代码与复杂度

1. 通用 DFS 代码框架

代码解释：DFSGraph
class DFSGraph(Graph):
    def __init__(self):
        super().__init__()
        self.time = 0

    def dfs(self):
        # 先把所有顶点初始化为未发现
        for aVertex in self:
            aVertex.setColor('white')
            aVertex.setPred(-1)

        # 图可能不连通，所以要从每个白色顶点尝试开始
        for aVertex in self:
            if aVertex.getColor() == 'white':
                self.dfsvisit(aVertex)

    def dfsvisit(self, startVertex):
        startVertex.setColor('gray')
        self.time += 1
        startVertex.setDiscovery(self.time)

        for nextVertex in startVertex.getConnections():
            if nextVertex.getColor() == 'white':
                nextVertex.setPred(startVertex)
                self.dfsvisit(nextVertex)

        startVertex.setColor('black')
        self.time += 1
        startVertex.setFinish(self.time)

2. 这段代码每一步在做什么

初始化颜色→遇到白点就开一棵 DFS 树→标灰并记发现时间→递归访问白色邻居→标黑并记结束时间
BFS 和 DFS 的关键区别：BFS 用队列显式保存“待访问顶点”；DFS 通过递归调用隐式使用了函数调用栈。

3. 复杂度

  • dfs() 中初始化和检查顶点，数量级是 O(|V|)。
  • dfsvisit() 中每条边最多被检查一次，数量级是 O(|E|)。
  • 合起来：O(|V| + |E|)。

例题2：DFS 为什么会生成森林？

题目：为什么通用 DFS 代码里，外层要对每个顶点都判断一次是否为白色？

答案：因为图可能不是连通的。从一个起点出发只能访问它能到达的部分；如果还有白色顶点，说明它属于另一块区域，需要再从它开始建立新的 DFS 树，所以整体可能是 DFS 森林。

四、拓扑排序：把依赖关系排成线性顺序

1. 拓扑排序解决什么问题

拓扑排序：把一个 DAG（有向无圈图）中的顶点排成一条线性序列，使得如果图中有边 v → w，那么在线性序列里 v 一定排在 w 前面。

适合的问题

课程先修关系、任务依赖、项目排期、工作流程。

不适合的问题

有循环依赖的问题。例如 A 依赖 B，B 又依赖 A，这时无法排出合法顺序。

2. DFS 实现拓扑排序

步骤做法原因
建图工作项/课程作为顶点，先后依赖作为有向边。边 v → w 表示 v 必须在 w 之前。
调用 DFS得到每个顶点的结束时间。一个顶点的后续依赖通常会更早完成探索。
排序输出按结束时间从大到小排列顶点。结束时间大的顶点排在前面。
代码解释：拓扑排序简化写法
def topological_sort(graph):
    graph.dfs()
    vertices = list(graph)
    vertices.sort(key=lambda v: v.getFinish(), reverse=True)
    return [v.getId() for v in vertices]

3. 小例子

一种合法顺序：买材料 → 热锅 → 调面糊 → 煎饼 → 吃。热锅和调面糊的相对顺序可以互换，只要都在煎饼之前即可。

例题3：判断拓扑序列是否合理

题目：若有边 A → C、B → C、C → D，序列 B, A, C, D 是否是合法拓扑排序？

答案：合法。A 和 B 都在 C 前面，C 在 D 前面；A 与 B 之间没有依赖，所以谁先谁后都可以。

五、强连通分支 SCC：有向图里的“互相可达群”

1. 什么是强连通分支

强连通分支 SCC：在有向图中，如果一个顶点集合 C 里任意两个顶点 v、w 都能互相到达，即 v 到 w 有路径，w 到 v 也有路径，而且 C 已经不能再扩大，那么 C 就是一个强连通分支。
只在有向图里重点讨论：因为无向图里“连通”本身就是双向的；强连通强调的是有向边限制下的“来回都通”。

2. 为什么有用

  • 网页链接网络中，可以发现高度聚集的网站群。
  • 大型有向图中，可以把互相可达的一组点压缩成一个点，从而简化图。
  • 强连通分支之间形成的压缩图通常更容易分析。

例题4：判断是否强连通

题目：若 A 能到 B，B 能到 C，C 能回到 A，那么 A、B、C 是否在同一个强连通分支中？

答案：是。因为 A、B、C 任意两点之间都可以沿着有向路径相互到达。

六、Kosaraju 算法：用两趟 DFS 找 SCC

1. 转置图是什么

图的转置：把有向图 G 中所有边方向反过来，得到 Gᵀ。例如原来有 v → w，转置后变成 w → v。
重要性质：原图 G 和转置图 Gᵀ 的强连通分支划分相同。因为如果一组点在原图里能互相到达，把所有边反过来后，仍然能互相到达。

2. Kosaraju 算法思路

步骤做法记忆
第一趟 DFS在原图 G 上运行 DFS，记录每个顶点的结束时间。先拿到“结束时间顺序”。
转置把 G 的所有边反向，得到 Gᵀ。把边全部倒过来。
第二趟 DFS在 Gᵀ 上按原图结束时间从大到小的顺序启动 DFS。按“最晚结束的先搜”。
得到结果第二趟 DFS 森林中的每棵树就是一个强连通分支。一棵树 = 一个 SCC。
代码解释：Kosaraju 的核心框架
def kosaraju_scc(graph):
    # 1. 原图 DFS，得到 finish 时间
    graph.dfs()
    order = sorted(list(graph), key=lambda v: v.getFinish(), reverse=True)

    # 2. 构造转置图
    gt = transpose(graph)

    # 3. 在转置图中按 order 开始 DFS
    # 每次从一个白色顶点开始搜到的一整棵树，就是一个 SCC
    sccs = []
    for v in gt.vertices_in_order(order):
        if v.getColor() == 'white':
            component = []
            dfs_collect(gt, v, component)
            sccs.append(component)
    return sccs
考试记忆：Kosaraju 不是“随便 DFS 两次”，而是：原图 DFS 记结束时间 → 转置图 → 按结束时间倒序 DFS。

例题5：Kosaraju 四步填空

题目：Kosaraju 算法先对原图做 DFS 计算什么？然后对什么图做第二趟 DFS？

答案：先计算每个顶点的结束时间；然后对转置图 Gᵀ 做第二趟 DFS，并按第一趟结束时间从大到小选择起点。

七、Dijkstra：带正权图的最短路径

1. 最短路径问题

带权最短路径：图中每条边都有权重，路径长度是边权重之和。Dijkstra 算法用于从一个起点出发，求到其它所有顶点的最短带权路径。

BFS 适合什么

无权图，或者所有边权相同。每走一条边代价一样。

Dijkstra 适合什么

带正权图。不同边的代价不同，需要累计权重。

2. Dijkstra 的核心变量

变量/结构含义作用
dist从起点到当前顶点的目前已知最短距离。作为优先级。
pred最短路径上当前顶点的前驱顶点。用于回溯路径。
优先队列按 dist 从小到大取顶点。每次先处理目前最近的顶点。
decreaseKey某顶点距离变小后，更新它在优先队列中的优先级。让更近的顶点更早出队。

3. 算法过程

起点 dist=0→其它点 dist=∞→取出 dist 最小点→用它更新邻接点→重复直到队列空
限制：课件强调 Dijkstra 只能用于正权重图；如果有负权边，就不应使用 Dijkstra。

例题6：手算一次 Dijkstra 更新

题目：从 A 出发，已知 dist[A]=0，有边 A→B 权重 4，A→C 权重 2。初始其它点为 ∞，第一次从 A 出队后，B 和 C 的 dist 变成多少？

答案：dist[B]=4，dist[C]=2。下一次应优先处理 C，因为 C 的 dist 更小。

八、Dijkstra 代码与复杂度

1. Python 简化实现

代码解释：用 heapq 写 Dijkstra
import heapq

def dijkstra(graph, start):
    # graph[u] = [(v, weight), ...]
    dist = {v: float('inf') for v in graph}
    pred = {v: None for v in graph}
    dist[start] = 0

    pq = [(0, start)]  # (当前距离, 顶点)

    while pq:
        cur_dist, u = heapq.heappop(pq)

        # 如果这条记录已经不是最新的，就跳过
        if cur_dist > dist[u]:
            continue

        for v, weight in graph[u]:
            new_dist = dist[u] + weight
            if new_dist < dist[v]:
                dist[v] = new_dist
                pred[v] = u
                heapq.heappush(pq, (new_dist, v))

    return dist, pred
代码里的松弛 relax：new_dist = dist[u] + weight。如果从起点到 u，再从 u 走到 v，比原来记录的 dist[v] 更短，就更新 dist[v] 和 pred[v]。

2. 复杂度

部分数量级原因
建堆/初始化O(|V|)每个顶点初始化一次。
顶点出队O(|V| log |V|)每个顶点出队一次，每次堆操作约 log |V|。
边的更新O(|E| log |V|)每条边可能引起一次优先级更新。
总复杂度O((|V|+|E|) log |V|)合并顶点和边的堆操作。

3. 用 pred 回溯路径

代码解释：从 pred 恢复 start 到 end 的路径
def shortest_path(pred, start, end):
    path = []
    cur = end
    while cur is not None:
        path.append(cur)
        if cur == start:
            break
        cur = pred[cur]
    path.reverse()
    return path

例题7：Dijkstra 和 BFS 的关系

题目：如果图中所有边权都相同，最短路径问题可以用什么算法理解？

答案：可以还原为 BFS。因为每条边代价一样，BFS 按层扩展时，第一次到达某顶点就是边数最少的路径。

九、最小生成树 MST：用最低总费用连通所有顶点

1. 信息广播问题为什么引出 MST

单播

广播源分别给每个接收者发送一份，可能在共享路由上重复传输同一消息。

洪水

每个路由器向所有邻居转发，若不限制会造成大量重复；通常要加 TTL。

最小生成树

只沿一棵覆盖所有顶点且总费用最小的树传播，既连通又避免重复圈。

2. 生成树与最小生成树

生成树：包含图中所有顶点，并用最少数量的边保持连通的子图。若有 |V| 个顶点，生成树有 |V|-1 条边。
最小生成树 MST：在所有生成树中，边权重之和最小的那棵树。
概念要求不能有什么
生成树包含所有顶点；连通；边数最少。不能有圈；有圈就说明边多了。
最小生成树在生成树基础上，总权重最小。不能为了省一条局部小边而破坏连通。
Dijkstra vs MST：Dijkstra 解决“从一个起点到各点的最短路径”；MST 解决“用总费用最小的一组边把所有点连起来”。目标不同，结果也可能不同。

例题8：生成树边数

题目：一个连通图有 6 个顶点，它的一棵生成树应该有多少条边？

答案：5 条。生成树若有 |V| 个顶点，就有 |V|-1 条边。

十、Prim 算法：每次接入最便宜的安全边

1. Prim 是贪心算法

Prim 算法：从一个起点开始维护一棵正在扩张的树 T。只要 T 还没包含所有顶点，就找一条权重最小且可以安全添加的边，把树外的新顶点接进来。
安全添加的边：一端在当前树 T 里，另一端不在 T 里。这样加进去不会形成圈，还能让树多包含一个顶点。

2. Prim 的过程

选起点→看从树内到树外的边→选权重最小的安全边→加入新顶点→直到包含所有顶点

3. Python 简化实现

代码解释：用优先队列实现 Prim
import heapq

def prim(graph, start):
    # graph[u] = [(weight, v), ...]
    visited = set()
    mst_edges = []
    pq = [(0, start, None)]  # (边权, 当前点, 从哪个点来)
    total = 0

    while pq and len(visited) < len(graph):
        weight, u, parent = heapq.heappop(pq)
        if u in visited:
            continue

        visited.add(u)
        total += weight
        if parent is not None:
            mst_edges.append((parent, u, weight))

        for w, v in graph[u]:
            if v not in visited:
                heapq.heappush(pq, (w, v, u))

    return total, mst_edges
代码理解：优先队列里放的是“把某个树外顶点接进来的候选边”。每次弹出最小边，如果终点已经在树里就跳过，否则加入 MST。

例题9：Prim 中什么边不能加

题目：当前树 T 已经包含 A、B、C。一条边连接 B 和 C，权重很小，这条边能加入 Prim 的树吗？

答案：不能。因为 B 和 C 都已经在树里，加这条边会形成圈；Prim 只能加“一端在树内，另一端在树外”的安全边。

十一、图算法综合对比

1. 本章图算法总表

算法解决问题关键数据/思想复杂度
DFS深度搜索、建 DFS 树/森林颜色、前驱、发现/结束时间、递归栈O(|V|+|E|)
拓扑排序DAG 的依赖顺序DFS 结束时间倒序O(|V|+|E|)
Kosaraju强连通分支原图 DFS、转置图、结束时间倒序 DFSO(|V|+|E|)
Dijkstra正权图单源最短路径dist、pred、优先队列、松弛O((|V|+|E|)log|V|)
Prim最小生成树贪心、安全边、优先队列常见堆实现约 O((|V|+|E|)log|V|)

2. 易混点集中对比

BFS / DFS

BFS 一层层扩展，适合无权最短路径；DFS 一条路尽量深入，适合拓扑排序、强连通分支等。

Dijkstra / Prim

Dijkstra 的优先级是“起点到当前点的累计距离”；Prim 的优先级是“把新点接进树的边权”。

拓扑排序 / SCC

拓扑排序要求 DAG；SCC 专门处理有向图中的互相可达分组。

最短路径 / 最小生成树

最短路径关注从一个源点到目标的路；最小生成树关注用最小总边权连通全图。

例题10：选算法

题目：课程 A 必须先于课程 B，课程 B 必须先于课程 C，要给课程安排学习顺序，用什么算法思想？

答案：拓扑排序。课程是顶点，先修关系是有向边，输出满足依赖的线性序列。

例题11：选算法

题目：给定城市道路图，每条路有通行时间，要求从学校到所有地点的最短时间，用什么算法？

答案：Dijkstra。因为这是带正权图上的单源最短路径问题。
