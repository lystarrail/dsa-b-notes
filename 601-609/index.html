树的所有代码都离不开两个字：递归。二叉堆则是“逻辑上像树，物理上用列表”。

二、什么是树：非线性的层次结构

1. 从线性结构到树结构

线性结构

像排队、链条、列表：一个接一个。一般每个数据项只有一个前驱和一个后继，首尾比较明显。

树结构

像文件夹、网页标签、家谱：一层套一层。每个节点有唯一父节点，但可以有多个子节点。

树最关键的直觉：不是“左右排队”，而是“上级—下级”的层次关系。

2. 树的三个重要特征

层次化
越靠近顶部越普遍，越靠近底部越具体。例如生物分类的界、门、纲、目、科、属、种。
子节点相互隔离
一个节点下面的子树，和另一个节点下面的子树互不影响。
根到节点路径唯一
可以用“从根到叶”的完整路径唯一标识一个对象，例如文件路径。

生活例子

文件系统就是树：C:\ 是根，下面有不同文件夹；每个文件有唯一的路径。HTML 文档也是树：html 下面有 head 和 body，标签一层层嵌套。

三、树结构术语与两个定义

1. 术语表：考试最容易直接考

术语通俗解释例子
节点 Node树里的一个数据单位，可以保存 key 或其它数据。文件夹、HTML 标签、表达式中的数字/符号。
边 Edge连接两个节点，表示父子关系。根到子节点的一条连线。
根 Root唯一没有父节点/入边的节点。文件系统中的根目录。
路径 Path由一串边连接起来的有序节点列表。HTML → BODY → UL → LI。
子节点 / 父节点一个节点向下连出去的是子节点；这个节点是它们的父节点。B、C 是根的子节点。
兄弟节点 Sibling拥有同一个父节点的节点。B 和 C 是兄弟。
子树 Subtree某个节点及其所有后代节点组成的小树。以 B 为根的整块结构。
叶节点 Leaf没有子节点的节点。D、E、F。
层级 Level从根到该节点经过的边数。根为 0，根的孩子为 1。
高度 Height整棵树中最大的层级。如果最深叶节点在第 3 层，高度为 3。

2. 树的定义：两种说法，其实是一回事

结构定义

  • 树由节点和边组成。
  • 有一个节点被指定为根。
  • 除根外，每个节点恰好有一个父节点。
  • 从根到每个节点的路径唯一。

递归定义

  • 树可以是空集。
  • 也可以由一个根节点和若干棵子树构成。
  • 每棵子树本身仍然是树。
二叉树：如果每个节点最多有两个子节点，这棵树就叫二叉树。这里的“最多两个”不是“必须两个”，可以只有左孩子、只有右孩子，也可以没有孩子。

四、树的嵌套列表实现

1. 用 Python list 表示二叉树

核心表示：[根节点值, 左子树, 右子树]。因为左子树和右子树也是树，所以它们也可以继续写成同样的三元列表。
a / \ b c / \ \ d e f
对应的嵌套列表
myTree = ['a',
          ['b', ['d', [], []], ['e', [], []]],
          ['c', [], ['f', [], []]]]

2. 基本操作代码

代码解释：三元列表法
def BinaryTree(r):
    # 创建只有根节点 r 的二叉树，左右子树先为空列表
    return [r, [], []]


def insertLeft(root, newBranch):
    # root[1] 是原来的左子树
    t = root.pop(1)
    if len(t) > 1:
        # 如果原来有左子树，新节点插在中间，旧左子树变成新节点的左子树
        root.insert(1, [newBranch, t, []])
    else:
        root.insert(1, [newBranch, [], []])
    return root


def insertRight(root, newBranch):
    t = root.pop(2)
    if len(t) > 1:
        root.insert(2, [newBranch, [], t])
    else:
        root.insert(2, [newBranch, [], []])
    return root


def getRootVal(root):
    return root[0]


def setRootVal(root, newVal):
    root[0] = newVal


def getLeftChild(root):
    return root[1]


def getRightChild(root):
    return root[2]
理解 insertLeft：如果当前位置本来就有左子树，直接覆盖会把旧数据弄丢。所以新节点要“插队”到当前位置和旧左子树之间，旧左子树被推下一层。

例题1：嵌套列表读树

题目：t = ['a', ['b', [], []], ['c', [], []]]，getRootVal(t)、getLeftChild(t)[0] 分别是什么？

答案：getRootVal(t) 是 'a'；getLeftChild(t)[0] 是左子树根节点 'b'。

五、树的节点链接实现

1. 用类表示节点

节点链接法：每个节点是一个对象，对象里保存自己的值 key，以及指向左子树、右子树的引用 leftChild 和 rightChild。
代码解释：BinaryTree 类
class BinaryTree:
    def __init__(self, rootObj):
        self.key = rootObj
        self.leftChild = None
        self.rightChild = None

    def insertLeft(self, newNode):
        if self.leftChild is None:
            self.leftChild = BinaryTree(newNode)
        else:
            t = BinaryTree(newNode)
            t.leftChild = self.leftChild
            self.leftChild = t

    def insertRight(self, newNode):
        if self.rightChild is None:
            self.rightChild = BinaryTree(newNode)
        else:
            t = BinaryTree(newNode)
            t.rightChild = self.rightChild
            self.rightChild = t

    def getRightChild(self):
        return self.rightChild

    def getLeftChild(self):
        return self.leftChild

    def setRootVal(self, obj):
        self.key = obj

    def getRootVal(self):
        return self.key

2. 嵌套列表法 vs 节点链接法

实现方式优点更适合
嵌套列表结构简单，直接体现递归；看小例子很直观。教学、表达式解析树的小规模实现。
节点链接类更像真正的数据结构；可以给节点添加更多方法和属性。复杂树结构、OJ 题、后续二叉查找树等。

例题2：画出代码生成的树

题目：执行下面代码后树长什么样？

r = BinaryTree('a')
r.insertLeft('b')
r.insertRight('c')
r.getLeftChild().insertLeft('d')

答案：

a / \ b c / d

六、表达式解析树：从全括号表达式建树

1. 为什么表达式可以变成树

表达式解析树：叶节点保存操作数，内部节点保存操作符。每棵子树都代表一个子表达式。

全括号表达式：

((7 + 3) * (5 - 2))

因为有括号，所以 7+3 和 5-2 必须先算，最后再算乘法。树的层次正好表达了这种优先级。

* / \ + - / \ / \ 7 3 5 2

2. 建树规则

读到的 token动作为什么
(创建左子节点，当前节点下降到左子节点。一个新的子表达式开始了。
操作符 + - * /把当前节点值设为操作符，创建右子节点，当前节点下降到右子节点。操作符需要左、右两个操作数。
操作数把当前节点值设为数字，当前节点上升到父节点。叶节点填完了，可以回到上一层。
)当前节点上升到父节点。一个子表达式结束了。
为什么要用栈？下降到子节点很容易，但普通树节点没有“回到父节点”的方法。下降前把当前节点压入栈，上升时弹出栈顶，就能找回父节点。

3. 建树代码

代码解释：用栈跟踪父节点
class Stack:
    def __init__(self):
        self.items = []
    def push(self, item):
        self.items.append(item)
    def pop(self):
        return self.items.pop()


def buildParseTree(fpexp):
    fplist = fpexp.split()
    pStack = Stack()
    eTree = BinaryTree('')
    pStack.push(eTree)
    currentTree = eTree

    for i in fplist:
        if i == '(':
            currentTree.insertLeft('')
            pStack.push(currentTree)
            currentTree = currentTree.getLeftChild()
        elif i in ['+', '-', '*', '/']:
            currentTree.setRootVal(i)
            currentTree.insertRight('')
            pStack.push(currentTree)
            currentTree = currentTree.getRightChild()
        elif i == ')':
            currentTree = pStack.pop()
        else:
            # i 是操作数
            currentTree.setRootVal(int(i))
            currentTree = pStack.pop()

    return eTree

pt = buildParseTree('( 3 + ( 4 * 5 ) )')

例题3：建树动作判断

题目：扫描 ( 3 + ( 4 * 5 ) )，读到 + 时应该做什么？

答案：把当前节点设为 +，创建右子节点，把当前节点下降到这个右子节点；同时把原当前节点压入栈，方便之后上升回来。

七、表达式解析树：递归求值与恢复表达式

1. 求值为什么天然递归

递归思路：如果是叶节点，值就是节点里的数字；如果是内部节点，就先递归算左子树和右子树，再用根节点的操作符把两个结果合起来。
算左子树→算右子树→用根操作符合并

2. 求值代码

代码解释：evaluate
import operator


def evaluate(parseTree):
    opers = {'+': operator.add,
             '-': operator.sub,
             '*': operator.mul,
             '/': operator.truediv}

    leftC = parseTree.getLeftChild()
    rightC = parseTree.getRightChild()

    if leftC and rightC:
        fn = opers[parseTree.getRootVal()]
        return fn(evaluate(leftC), evaluate(rightC))
    else:
        # 没有左右子节点，说明是叶节点，也就是操作数
        return parseTree.getRootVal()
为什么这是后序思想？求值顺序是先算左子树，再算右子树，最后处理根节点操作符，正好是“左—右—根”。

3. 从解析树恢复表达式

代码解释：用中序遍历生成全括号中缀表达式
def printexp(tree):
    sVal = ''
    if tree:
        if tree.getLeftChild() or tree.getRightChild():
            sVal = '('
        sVal = sVal + printexp(tree.getLeftChild())
        sVal = sVal + str(tree.getRootVal())
        sVal = sVal + printexp(tree.getRightChild())
        if tree.getLeftChild() or tree.getRightChild():
            sVal = sVal + ')'
    return sVal

答案：左子树值为 10，右子树值为 3，根节点做乘法，结果为 10 * 3 = 30。

八、树的三种遍历

1. 什么是遍历

遍历 Traversal：把树中所有节点都访问一遍。线性表按顺序走就行，但树有分支，所以必须规定访问顺序。

2. 三种遍历顺序

遍历方式顺序一句话记忆常见用途
前序 preorder根 → 左 → 右根在最前面，左在右前面复制树、打印章节目录。
中序 inorder左 → 根 → 右根在中间，左在右前面二叉搜索树有序输出；表达式恢复为中缀。
后序 postorder左 → 右 → 根根在最后，左在右前面表达式求值、删除树。
A / \ B C / \ \ D E F

前序：A B D E C F

中序：D B E A C F

后序：D E B F C A

3. 遍历代码

代码解释：只需要调整“访问根节点”的位置
def preorder(tree):
    if tree:
        print(tree.getRootVal())
        preorder(tree.getLeftChild())
        preorder(tree.getRightChild())


def inorder(tree):
    if tree:
        inorder(tree.getLeftChild())
        print(tree.getRootVal())
        inorder(tree.getRightChild())


def postorder(tree):
    if tree:
        postorder(tree.getLeftChild())
        postorder(tree.getRightChild())
        print(tree.getRootVal())

答案：D E B F C A。后序规则是先左子树、再右子树、最后根。

九、优先队列与二叉堆

1. 什么是优先队列

普通队列

按进入顺序排队，先进入的先出来，叫 FIFO。

优先队列

不是谁先进谁先出，而是谁优先级高谁先出。例如 VIP、操作系统重要任务。

二叉堆：实现优先队列的经典数据结构。它可以让插入 insert 和删除最小项 delMin 都保持在 O(log n)。

2. 二叉堆的两个性质

性质意思作用
结构性质：完全二叉树叶子最多只在最底层和次底层，最底层节点从左到右连续排列。让树接近平衡，高度约为 log n。
次序性质：堆次序最小堆中，父节点 key 小于等于子节点 key。保证根节点永远是最小项。

3. 完全二叉树为什么能放进列表

下标公式：列表第 0 项空着不用。若某节点下标是 p，左子节点下标是 2p，右子节点是 2p+1，父节点是 p//2。
列表形式
# 下标:  0  1  2   3   4   5   6   7
heap = [0, 5, 9, 11, 14, 18, 19, 21]
# 0 号位置不用，是为了让父子下标公式更简单

十、二叉堆的 Python 实现

1. BinaryHeap 类框架

代码解释：最小堆
class BinaryHeap:
    def __init__(self):
        self.heapList = [0]   # 第 0 项不用
        self.currentSize = 0

    def isEmpty(self):
        return self.currentSize == 0

    def size(self):
        return self.currentSize

    def findMin(self):
        if self.currentSize == 0:
            return None
        return self.heapList[1]

2. insert：先放末尾，再上浮

新 key 加到列表末尾→与父节点比较→比父节点小就交换上浮
代码解释：percUp + insert
def percUp(self, i):
    while i // 2 > 0:
        if self.heapList[i] < self.heapList[i // 2]:
            self.heapList[i], self.heapList[i // 2] = self.heapList[i // 2], self.heapList[i]
        i = i // 2


def insert(self, k):
    self.heapList.append(k)
    self.currentSize += 1
    self.percUp(self.currentSize)

3. delMin：移走根，再让新根下沉

保存堆顶最小值→最后一项移到根→沿较小孩子方向下沉
代码解释：percDown + minChild + delMin
def minChild(self, i):
    if i * 2 + 1 > self.currentSize:
        return i * 2
    else:
        if self.heapList[i * 2] < self.heapList[i * 2 + 1]:
            return i * 2
        else:
            return i * 2 + 1


def percDown(self, i):
    while (i * 2) <= self.currentSize:
        mc = self.minChild(i)
        if self.heapList[i] > self.heapList[mc]:
            self.heapList[i], self.heapList[mc] = self.heapList[mc], self.heapList[i]
        i = mc


def delMin(self):
    retval = self.heapList[1]
    self.heapList[1] = self.heapList[self.currentSize]
    self.currentSize -= 1
    self.heapList.pop()
    if self.currentSize > 0:
        self.percDown(1)
    return retval

4. buildHeap：从无序表一次建堆

代码解释：从最后一个非叶节点开始下沉
def buildHeap(self, alist):
    i = len(alist) // 2
    self.currentSize = len(alist)
    self.heapList = [0] + alist[:]
    while i > 0:
        self.percDown(i)
        i = i - 1
复杂度结论：insert 和 delMin 每次只沿一条从叶到根或从根到叶的路径移动，路径长度约为树高，所以单次都是 O(log n)。建堆要区分两种方法：逐个 insert 建堆是 O(n log n)；从中间往前调用 percDown 的 buildHeap 可以做到 O(n)。

5. 逐个 insert 建堆：为什么是 O(n log n)？

想象现在有 n 个乱序数字，要一个一个塞进原本为空的最小堆。每插入一个数字，先把它放到堆的最后一个位置，然后让它不断和父节点比较，必要时一路上浮。

  • 插入第 1 个元素：树高为 0，不需要移动。
  • 插入第 2 个元素：最多上浮 1 步。
  • 当堆已经接近完整时，后面大量新插入的节点都在底层，最多可能一路上浮到根，单次代价接近 log n。

严格地说，第 k 次插入的最坏代价是 O(log k)，所以总成本是 log 1 + log 2 + ... + log n = O(n log n)。可以把它理解成：很多靠后的节点都可能走一条接近树高的路。

n 次插入 × 每次最多 log n 步 ≈ O(n log n)

6. 从中间往前 buildHeap：为什么能做到 O(n)？

buildHeap 用的是反过来的思路：先把所有数字直接放进列表，形成一棵“结构已经完整”的完全二叉树，然后从最后一个非叶节点 n//2 开始，往前逐个调用 percDown 下沉。

  1. 最底层叶子节点：数量最多，约占整棵树的一半，但它们已经在底层，不需要下沉，移动 0 步。
  2. 倒数第二层节点：数量约 n/4，最多下沉 1 步。
  3. 再往上一层节点：数量约 n/8，最多下沉 2 步。
  4. 越靠近根：单个节点可能下沉更多步，但这样的节点数量越来越少，根节点虽然最多下沉 log n 步，但只有 1 个。

所以总步数不是 n 个节点都走 log n 步，而是“很多节点走很少步，少数节点走较多步”。

总步数 ≈ n/4×1 + n/8×2 + n/16×3 + ... + 1×log n = O(n)

这就是 Floyd 建堆算法的核心优势：从下往上局部调整，整体只需要线性时间。

例题6：二叉堆下标

题目：堆中某节点下标为 5，它的父节点下标是多少？如果它有孩子，左右孩子下标是多少？

答案：父节点是 5//2 = 2；左孩子是 10，右孩子是 11。

十一、OJ 题讲解与应用

1. OJ 901：二叉树复原 + 中序遍历

题目思想

给出层序列表，例如 [5, 4, 7, 3, None, 2, None, -1, None, 9]，要求恢复二叉树，然后进行中序遍历。

核心：建树用队列；中序遍历用递归。

代码解释：层序列表建树
from collections import deque

class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None


def build_tree_level(data):
    if not data or data[0] is None:
        return None

    root = TreeNode(data[0])
    q = deque([root])
    i = 1

    while q and i < len(data):
        node = q.popleft()

        if i < len(data) and data[i] is not None:
            node.left = TreeNode(data[i])
            q.append(node.left)
        i += 1

        if i < len(data) and data[i] is not None:
            node.right = TreeNode(data[i])
            q.append(node.right)
        i += 1

    return root


def inorder_node(root, ans=None):
    if ans is None:
        ans = []
    if root:
        inorder_node(root.left, ans)
        ans.append(root.val)
        inorder_node(root.right, ans)
    return ans

2. OJ 902：翻转二叉树

题目思想

把每个节点的左子树和右子树交换。例：根 4 的左右子树 2 和 7 交换，下面所有节点也递归交换。

核心：翻转二叉树用递归；建树和中序遍历方法与前题相同。

代码解释：递归翻转
def invert_tree(root):
    if root is None:
        return None

    # 先递归翻转左右子树，再交换
    left = invert_tree(root.left)
    right = invert_tree(root.right)
    root.left, root.right = right, left
    return root

3. OJ 903：多叉树后序遍历

题目思想

多叉树不是每个节点最多两个孩子，而是可以有多个孩子。可以用嵌套列表表示：[节点值, [子树1, 子树2, ...]]。

核心：后序遍历仍然是递归：先遍历所有子树，最后访问根节点。

代码解释：多叉树后序遍历
def postorder_nary(tree):
    # tree = [value, children]
    value, children = tree[0], tree[1]
    ans = []
    for child in children:
        ans.extend(postorder_nary(child))
    ans.append(value)
    return ans

# 示例：1 的孩子是 2、7、8
root = [1, [[2, []], [7, []], [8, []]]]
print(postorder_nary(root))  # [2, 7, 8, 1]

4. 自主练习思路

题型核心方法一句话思路
合并二叉树递归两个节点都存在就相加；只有一个存在就返回那个节点。
对称二叉树递归/层序判断左子树的左与右子树的右、左子树的右与右子树的左是否相等。
前序 + 中序推理二叉树递归前序第一个是根；在中序中找到根，左边是左子树，右边是右子树，然后递归。
代码解释：由前序和中序复原二叉树
def build_from_pre_in(preorder, inorder):
    if not preorder:
        return None

    root_val = preorder[0]
    root = TreeNode(root_val)
    mid = inorder.index(root_val)

    root.left = build_from_pre_in(preorder[1:1+mid], inorder[:mid])
    root.right = build_from_pre_in(preorder[1+mid:], inorder[mid+1:])
    return root
