二、508 什么是散列：用 key 直接算位置

1. 从“找数据”变成“算位置”

散列 Hashing：通过一个散列函数，把数据项或关键码 key 转换成散列表中的槽号。查找时不用从头扫描，而是直接去这个槽看看。
数据项 / key→hash(key)→槽号 slot→存 / 查
小白理解：假设宿舍快递柜有 0 到 10 号格子。普通查找像一个柜子一个柜子翻；散列像先用手机号算出“应该在 4 号柜”，然后直接去 4 号柜。

2. 基本术语

术语意思记忆方式
散列表 hash table专门为了快速定位和查找设计的数据集合一排有编号的柜子
槽 slot散列表中的一个存储位置，每个槽有唯一编号某个柜门编号
散列函数 hash function把 key/data 转换成槽号的函数算柜号的规则
空槽 None还没有存放数据的位置柜子里是空的

3. 为什么说理想情况下是 O(1)

O(1) 的意思：不是“只用 1 秒”，而是“数据规模变大时，查找步骤数量基本不跟着变大”。如果散列函数直接算出位置，并且没有冲突，那么查一次只需要算槽号和看这个槽。
但不要背成绝对 O(1)：散列查找的性能依赖于散列函数和冲突情况。如果冲突很多，查找可能要继续探测或查链表，就不再那么简单。

3. 负载因子 λ

负载因子：λ = 数据项个数 / 槽数。例如上面 6 个数据项，占 11 个槽中的 6 个，负载因子就是 6/11。
理解：λ 小，柜子空得多，冲突少；λ 大，柜子快满了，冲突多，找空槽也更麻烦。

四、509 完美散列函数与数据指纹

1. 完美散列函数

完美散列函数：对于给定的一组数据，如果每个数据项都能被映射到不同的槽里，就叫完美散列函数。
为什么不总用完美散列？固定数据集可以想办法设计，但实际数据经常变化，很难始终保证不冲突。如果为了手机号这种巨大范围直接开百亿个槽，空间浪费太大。

2. 好的散列函数应满足什么

要求解释为什么重要
冲突少不同 key 尽量去不同槽冲突越少，查找越接近 O(1)
计算快hash(key) 不能太费时间散列函数本身太慢，会抵消查找优势
分布均匀数据项不要集中挤到少数槽避免某些槽附近过度聚集

3. MD5/SHA：把数据变成“指纹”

MD5、SHA 系列是著名的近似完美散列函数。它们把任意长度的数据变成固定长度的摘要，常用于文件校验、密码散列保存、防篡改。
代码解释：Python hashlib 生成摘要
import hashlib

# Python 3 中 hashlib 需要 bytes，所以要 encode()
text = "hello world!".encode("utf-8")
print(hashlib.md5(text).hexdigest())
print(hashlib.sha1(text).hexdigest())

# 大文件可以分块 update，避免一次性读入内存
m = hashlib.md5()
for part in [b"hello world!", b"this is part #2", b"this is part #3"]:
    m.update(part)
print(m.hexdigest())
注意：这里的 MD5/SHA 是散列摘要，不是“压缩包”。摘要通常不能还原原文；只保存密码摘要也比保存明文密码更安全。

五、510 区块链：散列的应用

1. 只抓和本章有关的核心

区块链：可以看成一个由很多区块串起来的分布式数据库。每个区块记录数据，并在区块头里保存前一个区块的散列值。

2. 为什么不容易篡改

  • 散列有抗修改性：数据改一点，散列值会明显改变。
  • 每个区块保存前一个区块的散列值：改一个区块，会影响后续链条。
  • 工作量证明 POW：要找到满足条件的 Nonce 只能暴力尝试，修改大量区块成本极高。
本章需要掌握到什么程度：知道区块链是散列的应用，知道“前一区块散列值 + 抗修改性 + 工作量证明”让篡改变得困难即可，不需要背比特币价格和矿机细节。

六、511 散列函数设计

1. 折叠法 folding method

折叠法：把数字按位数切成几段，把几段相加，再对表大小求余。
62767255→62, 76, 72, 55→相加 265→265 % 11 = 1

例题2：折叠法

题目：用折叠法对电话号码 62767255 散列，表大小为 11，两位一段，求散列值。

答案：62+76+72+55=265，265%11=1，所以 h(62767255)=1。

变式：隔数反转成 62、67、72、55，则 62+67+72+55=256，256%11=3。

2. 平方取中法 mid-square method

平方取中法：先把数据项平方，取平方结果的中间若干位，再对表大小求余。

例题3：平方取中法

题目：用平方取中法对 44 散列，表大小为 11，取平方数中间两位。

答案：44×44=1936，中间两位是 93，93%11=5，所以散列值为 5。

3. 字符串散列

非数字数据也能散列。常用做法是把字符转换成整数，例如 Python 中 ord('c') 得到字符 c 的编码。
代码解释：直接把每个字符 ord 值相加
def hash(astring, tablesize):
    total = 0
    for pos in range(len(astring)):
        total = total + ord(astring[pos])
    return total % tablesize

print(hash("cat", 11))  # 4，因为 99+97+116 = 312，312%11 = 4
问题：如果只求字符编码之和，cat、act、tac 的总和一样，散列值也一样，这对所有变位词都会冲突。
代码解释：加入位置权重，降低变位词冲突
def hash_weighted(astring, tablesize):
    total = 0
    for pos in range(len(astring)):
        total = total + (pos + 1) * ord(astring[pos])
    return total % tablesize

print(hash_weighted("cat", 11))  # 3，因为 99*1+97*2+116*3 = 641

例题4：字符串散列

题目：表大小为 11，对字符串 cat 做散列。已知 ord('c')=99，ord('a')=97，ord('t')=116。

答案1：直接求和：99+97+116=312，312%11=4。

答案2：带位置权重：99×1+97×2+116×3=641，641%11=3。

七、512 冲突解决：开放定址与线性探测

1. 开放定址 open addressing

开放定址：如果原本应该放的槽已经被占用，就在散列表内部继续寻找另一个空槽。

2. 线性探测 linear probing

线性探测：从冲突槽开始，向后一个一个槽寻找空槽；到表尾还没找到，就从表头继续找。

例题5：线性探测插入 44、55、20

已知：表大小为 11，已经有 77 在 0 号槽，26 在 4 号槽，93 在 5 号槽，17 在 6 号槽，31 在 9 号槽，54 在 10 号槽。散列函数 h(item)=item%11。

  • 插入 44：44%11=0，0 号槽被 77 占，向后找，1 号槽空，放入 1。
  • 插入 55：55%11=0，0 号槽被占，1 号也被 44 占，2 号槽空，放入 2。
  • 插入 20：20%11=9，9 号槽被 31 占，10 号被 54 占，0、1、2 已占，3 号槽空，放入 3。

3. 查找也要按同样规则

为什么查找不能只看原槽？例如找 55，55%11=0，但 0 号槽是 77。因为插入时可能发生过线性探测，所以查找也必须从 0 号槽继续向后找，直到找到 55 或遇到空槽。
线性探测缺点：容易形成聚集。越聚集，插入和查找越像局部顺序查找，效率下降。

八、再散列与数据链法

1. 再散列 rehashing

再散列：冲突后重新计算下一个尝试槽号。线性探测可写成 rehash(pos)=(pos+1)%size。
探测方式公式/规则特点
线性探测(pos+1)%size简单，但容易聚集
跳跃式探测(pos+skip)%size跳着找，能缓解聚集
二次探测尝试 h,h+1,h+4,h+9,h+16...步长逐渐变化，减少聚集
skip 取值提醒：跳跃式探测中，skip 不能与表大小形成不好的周期，否则可能有些空槽永远探测不到。把散列表大小设成素数是一种常见技巧。

2. 数据链法 chaining

数据链法：每个槽不只放一个数据，而是放一个集合或链表。冲突时，直接把数据项加入这个槽对应的链。
对比：开放定址是“冲突后在表内找别的槽”；数据链法是“冲突项都挂在原槽下面”。数据链法允许 λ 大于 1，因为一个槽能放多个数据项。

九、513 映射 ADT Map 与 Python 实现

1. ADT Map 是什么

映射 Map：保存 key-data 键值关联的无序集合。key 具有唯一性，通过 key 可以唯一确定一个 data。Python 的字典 dict 就是常见的 Map。
操作含义
Map()创建空映射
put(key,val)加入 key-val；如果 key 已存在，则替换旧值
get(key)给定 key 返回 value；不存在则返回 None
del map[key]删除 key-value 关联
len()返回键值对数量
key in map判断 key 是否存在

2. 用两个平行列表实现 HashTable

代码核心：slots 列表存 key；data 列表存 value。某个 key 在 slots 的第 i 个位置，它的 value 就在 data 的第 i 个位置。
完整代码解释：HashTable 的 put/get 与 [] 访问
class HashTable:
    def __init__(self):
        self.size = 11                  # 常用素数大小，便于探测覆盖更多槽
        self.slots = [None] * self.size # 保存 key
        self.data = [None] * self.size  # 保存 value

    def hashfunction(self, key):
        return key % self.size

    def rehash(self, oldhash):
        return (oldhash + 1) % self.size # 线性探测：向后找下一个槽

    def put(self, key, data):
        hashvalue = self.hashfunction(key)

        if self.slots[hashvalue] is None:
            # 情况1：目标槽为空，直接放
            self.slots[hashvalue] = key
            self.data[hashvalue] = data
        else:
            if self.slots[hashvalue] == key:
                # 情况2：key 已经存在，替换 value
                self.data[hashvalue] = data
            else:
                # 情况3：发生冲突，继续 rehash 找空槽或同一个 key
                nextslot = self.rehash(hashvalue)
                while self.slots[nextslot] is not None and                       self.slots[nextslot] != key:
                    nextslot = self.rehash(nextslot)

                if self.slots[nextslot] is None:
                    self.slots[nextslot] = key
                    self.data[nextslot] = data
                else:
                    self.data[nextslot] = data

    def get(self, key):
        startslot = self.hashfunction(key)
        data = None
        stop = False
        found = False
        position = startslot

        while self.slots[position] is not None and not found and not stop:
            if self.slots[position] == key:
                found = True
                data = self.data[position]
            else:
                position = self.rehash(position)
                if position == startslot: # 绕回起点，说明表里没有这个 key
                    stop = True
        return data

    def __getitem__(self, key):
        return self.get(key)

    def __setitem__(self, key, data):
        self.put(key, data)

H = HashTable()
H[54] = "cat"
H[26] = "dog"
H[93] = "lion"

情况代码判断动作

key 已存在self.slots[hashvalue] == key替换 data

4. get 方法的停止条件

get 从 hashfunction(key) 得到查找起点。如果找到 key，返回 data；如果遇到空槽，说明不存在；如果绕一圈回到起点，也要停止，防止死循环。

十、算法分析与例题应用

1. 散列算法分析

散列最理想的结论：冲突少、负载因子小、散列函数好时，查找可接近 O(1)。但冲突存在时，比较次数取决于负载因子和冲突解决方案。
冲突解决方式成功查找平均比较次数失败查找平均比较次数说明
线性探测开放定址1/2 × (1 + 1/(1-λ))1/2 × (1 + (1/(1-λ))²)λ 在 0 到 1 之间；λ 越接近 1 越糟
数据链法1 + λ/2λλ 可以大于 1；链越长越慢
不要硬背公式：考试更常考理解：负载因子越大，冲突越多；开放定址表快满时很危险；数据链法链越长，查找越像在小链表中顺序查找。

2. 课堂练习应用：负载因子达到阈值时自动扩容

例题6：重新实现 put，使散列表能自动调整大小

题目：请设计算法重新实现散列表的 put 方法，使散列表可以在负载因子达到一个预设值时自动调整大小。要求扩容后原有数据仍能正常访问。

思路：不能只把旧数组原样复制过去，因为表大小变了，key%size 的结果也会变。正确做法是：新建更大的表，然后把旧 key-data 逐个重新 put 进去，也就是重新散列。

伪代码解释：扩容后必须重新散列旧数据
def put(self, key, data):
    # 1. 如果插入后负载因子超过阈值，先扩容
    if (self.count + 1) / self.size > self.max_load:
        old_slots = self.slots
        old_data = self.data
        self.size = next_prime(self.size * 2)
        self.slots = [None] * self.size
        self.data = [None] * self.size
        self.count = 0

        # 2. 注意：不能直接复制旧数组，必须按新 size 重新散列
        for i in range(len(old_slots)):
            if old_slots[i] is not None:
                self.put(old_slots[i], old_data[i])

    # 3. 再按普通 put 逻辑插入新 key-data
    ordinary_put_without_resize(self, key, data)

3. 本章常见判断题/选择题思路

题目问法关键答案
散列为什么比二分查找还快？二分查找利用有序关系缩小范围；散列直接计算位置。
冲突是不是错误？不是。冲突很常见，需要系统的冲突解决方案。
为什么表大小常设为素数？有助于跳跃探测覆盖更多槽，减少不良周期。
HashTable 为什么用两个列表？一个存 key，一个存 value；相同下标形成关联。
为什么不能无限追求复杂散列函数？散列函数太慢，查找优势会被计算开销抵消。
