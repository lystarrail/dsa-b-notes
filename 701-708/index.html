二、图的基本概念

1. 图是什么

图 Graph：由一组顶点和一组边构成。顶点表示对象，边表示对象之间的关系。树也是图的一种特殊情况，但图比树更自由，边可以更复杂，也可能有圈。

现实问题

  • 公交站点和线路：站点是顶点，线路是边。
  • 网页和超链接：网页是顶点，链接是边。
  • 课程先修关系：课程是顶点，先修关系是有向边。
  • 社交网络：人是顶点，好友关系是边。

2. 树和图的区别

结构特点例子
树层次明显；通常有唯一根；从根到节点路径唯一；无圈。文件系统、HTML 标签树。
图更一般；顶点之间可以任意连接；可以有方向、权重、圈。交通网络、互联网、社交网络。
理解：树更像“组织架构图”，图更像“地图”。地图里两个地点可能有多条路相连，也可能绕一圈回到原地，这就是图比树更一般的地方。

三、图结构术语与定义

1. 术语表

术语通俗解释例子
顶点 Vertex / Node图里的点，通常有 key，也可以带 payload。公交站、网页、单词、棋盘格。
边 Edge / Arc连接两个顶点，表示关系。可以有方向。从站点 A 到站点 B 的线路。
无向图边没有方向，A 连 B 就等于 B 连 A。普通好友关系。
有向图边有方向，A → B 不一定能 B → A。网页超链接、课程先修关系。
权重 Weight边上的代价。距离、通行时间、票价。
路径 Path由边依次连接起来的顶点序列。v3 → v4 → v0 → v1。
圈 Cycle首尾顶点相同的路径。v5 → v2 → v3 → v5。
DAGDirected Acyclic Graph，有向无圈图。课程先修关系通常适合表示成 DAG。

2. 图的形式化定义

定义：一个图可以写成 G = (V, E)。其中 V 是顶点集合，E 是边集合。若是带权图，边可以写成 (v, w, weight)。

例题1：路径长度怎么算

题目：路径 (v3, v4, v0, v1) 包含边 (v3,v4,7)、(v4,v0,1)、(v0,v1,5)。它的无权长度和带权长度分别是多少？

答案：无权长度看边数，是 3；带权长度看权重和，是 7+1+5=13。

四、ADT Graph 与两种存储表示

1. ADT Graph 操作

操作作用
Graph()创建空图。
addVertex(vert)向图中加入顶点。
addEdge(fromVert, toVert)添加有向边。
addEdge(fromVert, toVert, weight)添加带权有向边。
getVertex(vKey)按 key 查找顶点。
getVertices()返回所有顶点 key。
vert in graph判断顶点是否存在。

2. 邻接矩阵 Adjacency Matrix

邻接矩阵：用一个二维矩阵表示连接关系。矩阵的行和列都代表顶点；如果从行顶点到列顶点有边，就在对应格子填 1 或权重。

优点：简单，查两个点是否相连很直接。

缺点：空间通常是 O(|V|²)，如果边很少，会浪费很多空格子。

3. 邻接表 Adjacency List

邻接表：维护一个顶点主表；每个顶点再保存自己连向哪些顶点，以及边的权重。

优点：只保存实际存在的边，适合稀疏图。

缺点：如果要快速判断“任意两个点是否相连”，不如矩阵直观。

表示方法空间特点适合场景
邻接矩阵O(|V|²)顶点少、边很多、常查两点是否相连。
邻接表接近 O(|V|+|E|)边远少于 |V|² 的稀疏图。

例题2：词梯图为什么适合邻接表

题目：有 5110 个四字母单词。若用邻接矩阵，需要约 5110×5110=26112100 个单元；实际边约 53286 条。应该选邻接矩阵还是邻接表？

答案：选邻接表。因为实际边只占矩阵规模很小一部分，是典型稀疏图；邻接矩阵会浪费大量空间。

五、Graph 的 Python 实现

1. Vertex 类：一个顶点知道自己连向谁

代码解释：Vertex 顶点类
class Vertex:
    def __init__(self, key):
        self.id = key
        self.connectedTo = {}   # 邻接顶点 -> 权重

    def addNeighbor(self, nbr, weight=0):
        self.connectedTo[nbr] = weight

    def __str__(self):
        return str(self.id) + ' connectedTo: ' + str([x.id for x in self.connectedTo])

    def getConnections(self):
        return self.connectedTo.keys()

    def getId(self):
        return self.id

    def getWeight(self, nbr):
        return self.connectedTo[nbr]
理解：connectedTo 是一个字典。字典的 key 是“邻接顶点对象”，value 是“到这个邻接顶点的边权重”。这就是邻接表思想的 Python 版本。

2. Graph 类：保存所有顶点的主表

代码解释：Graph 图类
class Graph:
    def __init__(self):
        self.vertList = {}       # key -> Vertex 对象
        self.numVertices = 0

    def addVertex(self, key):
        self.numVertices += 1
        newVertex = Vertex(key)
        self.vertList[key] = newVertex
        return newVertex

    def getVertex(self, n):
        if n in self.vertList:
            return self.vertList[n]
        return None

    def __contains__(self, n):
        return n in self.vertList

    def addEdge(self, f, t, cost=0):
        if f not in self.vertList:
            self.addVertex(f)
        if t not in self.vertList:
            self.addVertex(t)
        self.vertList[f].addNeighbor(self.vertList[t], cost)

    def getVertices(self):
        return self.vertList.keys()

    def __iter__(self):
        return iter(self.vertList.values())

例题3：addEdge 会做什么

题目：执行 g.addEdge(0, 1, 5) 时，如果顶点 0 或 1 还不存在，会发生什么？

答案：addEdge 会先检查顶点是否存在。不存在就先 addVertex 创建顶点，然后在顶点 0 的邻接表中加入顶点 1，权重为 5。

六、词梯问题：把单词变换转成图

1. 词梯问题是什么

词梯 Word Ladder：从一个单词变到另一个单词，每一步只能改变一个字母，并且中间结果都必须是合法单词。目标通常是找最短变换序列。

示例

FOOL → POOL → POLL → POLE → PALE → SALE → SAGE

每一步都只改了一个字母，所以这是一条从 FOOL 到 SAGE 的有效路径。

2. 建图方式

单词作为顶点→只差 1 个字母就连边→找最短路径
直接比较太慢：如果把每个单词和其它所有单词都比较，复杂度接近 O(n²)。5110 个单词就要超过 2600 万次比较。

3. 桶算法：用通配符加速建图

桶思想：对四字母单词，每次用 _ 替换一个位置，得到桶标签。落在同一个桶里的单词，说明它们只差被替换的那个位置，因此可以互相连边。
代码解释：构建词梯图
def buildGraph(wordFile):
    d = {}
    g = Graph()

    # 第一步：把单词放入对应的桶
    with open(wordFile, 'r', encoding='utf-8') as wfile:
        for line in wfile:
            word = line.strip()
            for i in range(len(word)):
                bucket = word[:i] + '_' + word[i+1:]
                d.setdefault(bucket, []).append(word)

    # 第二步：同一个桶内的单词两两连边
    for bucket in d.keys():
        for word1 in d[bucket]:
            for word2 in d[bucket]:
                if word1 != word2:
                    g.addEdge(word1, word2)
    return g

例题4：桶标签判断

题目：FOOL 会进入哪些桶？FOOL 和 POOL 为什么会连边？

答案：FOOL 会进入 _OOL、F_OL、FO_L、FOO_ 四个桶。FOOL 和 POOL 都在 _OOL 桶里，说明只差第一个字母，所以可以连边。

七、BFS 广度优先搜索

1. BFS 的核心直觉

BFS：从起点开始，先找距离 1 的所有顶点，再找距离 2 的所有顶点，再找距离 3 的所有顶点……它是一层一层向外扩散。

2. BFS 需要记录什么

属性含义作用
color白色未发现；灰色已发现；黑色探索完成。避免重复访问。
distance从起点到当前顶点的边数。记录最短距离。
predecessor当前顶点是从哪个顶点发现的。反向追踪最短路径。
Queue保存已经发现但尚未完全探索的顶点。保证先发现的顶点先扩展，实现逐层搜索。

3. BFS 流程

  1. 把起点设为灰色，距离为 0，前驱为 None，入队。
  2. 队列不空时，取出队首顶点作为当前顶点。
  3. 遍历当前顶点的所有邻接点。
  4. 如果邻接点是白色，说明第一次发现它：设为灰色，距离 = 当前距离 + 1，前驱 = 当前顶点，入队。
  5. 当前顶点的邻接点检查完后，把当前顶点设为黑色。

例题5：BFS 为什么能保证最短词梯

题目：词梯图是无权图，为什么从 FOOL 开始 BFS，第一次发现 SAGE 时得到的一定是最短变换序列？

答案：BFS 按距离逐层扩展。第一次发现 SAGE 时，所有更短距离的可能顶点已经被检查过，因此这条路径的边数最少。

八、BFS 代码与复杂度

1. BFS 代码

代码解释：BFS 主体
from collections import deque

def bfs(g, start):
    # 初始化起点
    start.setDistance(0)
    start.setPred(None)
    start.setColor('gray')

    vertQueue = deque()
    vertQueue.append(start)

    while vertQueue:
        currentVert = vertQueue.popleft()
        for nbr in currentVert.getConnections():
            if nbr.getColor() == 'white':
                nbr.setColor('gray')
                nbr.setDistance(currentVert.getDistance() + 1)
                nbr.setPred(currentVert)
                vertQueue.append(nbr)
        currentVert.setColor('black')
说明：这里假设 Vertex 类已经额外实现了 setColor/getColor、setDistance/getDistance、setPred/getPred 这些方法。它们是为了 BFS 服务的辅助属性。

2. 根据前驱回溯路径

代码解释：从目标顶点往回追到起点
def traverse(y):
    x = y
    path = []
    while x is not None:
        path.append(x.getId())
        x = x.getPred()
    path.reverse()
    return path

3. 复杂度

BFS 本体复杂度：O(|V| + |E|)。

  • 每个顶点最多入队、出队一次，所以顶点部分是 O(|V|)。
  • 遍历邻接表时，每条边最多被检查一次，所以边部分是 O(|E|)。

词梯问题整体还包括：

  • 回溯路径最多走过所有顶点，最坏 O(|V|)。
  • 建图如果采用暴力两两比较，最多 O(|V|²)；采用桶算法会更实际。

九、骑士周游：把棋盘移动转成图

1. 问题是什么

骑士周游：国际象棋里的骑士按“马走日”移动。从一个格子出发，要求走遍棋盘上所有格子，且每个格子恰好访问一次。

2. 建图方式

棋盘格作为顶点→合法马步作为边→寻找覆盖所有顶点的路径

3. 建骑士图代码

代码解释：生成每个格子的合法移动边
def posToNodeId(row, col, board_size):
    return row * board_size + col


def legalCoord(x, board_size):
    return 0 <= x < board_size


def genLegalMoves(x, y, board_size):
    newMoves = []
    moveOffsets = [(-1, -2), (-1, 2), (-2, -1), (-2, 1),
                   (1, -2), (1, 2), (2, -1), (2, 1)]
    for dx, dy in moveOffsets:
        newX = x + dx
        newY = y + dy
        if legalCoord(newX, board_size) and legalCoord(newY, board_size):
            newMoves.append((newX, newY))
    return newMoves


def knightGraph(board_size):
    ktGraph = Graph()
    for row in range(board_size):
        for col in range(board_size):
            nodeId = posToNodeId(row, col, board_size)
            for newPos in genLegalMoves(row, col, board_size):
                nid = posToNodeId(newPos[0], newPos[1], board_size)
                ktGraph.addEdge(nodeId, nid)
    return ktGraph

例题6：棋盘格如何变成顶点编号

题目：如果棋盘大小为 8，格子 (row=2, col=3) 的顶点编号是多少？

答案：2*8+3=19。这种编号方式可以把二维棋盘格转成一维顶点 key。

十、DFS 回溯与启发式

1. DFS 的定义：深度优先搜索

DFS（Depth First Search，深度优先搜索）：从一个起点出发，先沿着一条分支尽量往深处走；如果走到无法继续、还没有得到目标结果，就回退到上一层，换另一条分支继续搜索。
关键词含义
尽量深入不是先把同一层都看完，而是先沿一条路走到底。
回溯这条路走不通时，撤销当前选择，回到上一层试别的路。
路径记录骑士周游中需要记录当前已经走过的格子，通常用列表/栈保存路径。
和 BFS 的关系：BFS 像水波纹一样一层层扩散，适合无权最短路径；DFS 像走迷宫，先沿一条路深入，走不通再退回来，适合搜索一条完整可行路径。

2. DFS 和 BFS 的直觉区别

BFS

一层一层扩展，先发现距离近的点，所以适合找无权图最短路径。

DFS

沿一条路尽量走深，走不通再回退，适合搜索完整路径或所有可能。

3. 骑士周游中的 DFS 回溯

回溯思路：当前顶点加入路径并标记为已访问；尝试每个还没访问过的邻接顶点；如果某条路最终失败，就撤销标记，把当前顶点从路径里弹出，换另一条路试。
代码解释：骑士周游 DFS
def knightTour(n, path, u, limit):
    # n 表示当前深度，path 记录路径，u 是当前顶点
    u.setColor('gray')
    path.append(u)

    if n < limit:
        nbrList = list(u.getConnections())
        i = 0
        done = False
        while i < len(nbrList) and not done:
            if nbrList[i].getColor() == 'white':
                done = knightTour(n + 1, path, nbrList[i], limit)
            i += 1
        if not done:
            # 本分支失败，撤销选择，回到上一层
            path.pop()
            u.setColor('white')
    else:
        done = True

    return done

4. 复杂度和 Warnsdorff 启发式

初始 DFS 很慢：骑士周游搜索树层数接近棋盘格数，复杂度可看作指数级。资料中提到 5×5 还能较快求解，8×8 可能耗时很长。
Warnsdorff 规则：选择下一步时，优先走向“后续合法移动位置最少”的格子。这样可以先处理最容易被堵死的位置，大幅减少回溯次数。
代码解释：按后续可走数量排序
def orderByAvail(n):
    resList = []
    for v in n.getConnections():
        if v.getColor() == 'white':
            c = 0
            for w in v.getConnections():
                if w.getColor() == 'white':
                    c += 1
            resList.append((c, v))
    resList.sort(key=lambda x: x[0])
    return [y[1] for y in resList]
理解：这不是改变问题本身，而是改变“先试哪条路”的顺序。它属于启发式规则：不保证从理论上把指数复杂度变成多项式，但能极大改善实际运行速度。
