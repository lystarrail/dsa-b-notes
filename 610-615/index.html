二、二叉查找树 BST：左小右大

1. BST 的定义

二叉查找树 Binary Search Tree：对任意节点来说，左子树里的 key 都比它小，右子树里的 key 都比它大。这样查找时就像二分查找一样，每次能排除一边。

2. 插入顺序会改变 BST 形状

同一组 key，只要插入顺序不同，生成的 BST 形状就可能不同。下面两种顺序使用的是同一批数字：

顺序 A：70, 31, 93, 94, 14, 23, 73

23 沿路径比较：23 < 70，23 < 31，23 > 14，所以在 14 的右边。

顺序 B：14, 23, 31, 70, 73, 93, 94

升序插入时，每个新 key 都一直走右子树，BST 会退化成一条链。

结论：普通 BST 的性能不只和数据个数有关，也和插入顺序有关；树越歪，查找和插入越慢。

3. 为什么中序遍历能得到升序

左子树→根节点→右子树

BST 中对任意节点来说，左子树都小于该节点，右子树都大于该节点。中序遍历正好先访问左子树，再访问根，再访问右子树，所以结果是升序。

例题1：为什么给 BST 填值时常用中序遍历？

题目思想：给定一个固定的二叉树结构和一组整数，把整数填入节点，使它变成 BST。

解题关键：先对树结构做中序遍历得到“节点访问顺序”，再把整数排序后按这个顺序填进去。因为 BST 的中序遍历必须是升序。

三、BST 的节点结构与 ADT Map

1. BST 为什么能实现 ADT Map

ADT Map：保存 key-data 键值对，通过 key 唯一确定一个 data。BST 可以把 key 存在树节点中，payload 存对应数据；查找 key 时按大小关系走左/右子树。
操作含义BST 中怎么做
put(key, val)加入或更新键值对按 key 大小找到插入位置
get(key)根据 key 取 val从根出发比较，向左/右找
del map[key]删除 key 对应节点找到节点后按三种情况 remove
key in map判断 key 是否存在本质是 get/_get 查找

2. TreeNode 类：一个节点要记住什么

代码解释：TreeNode 不是只存一个值，还要存连接关系
class TreeNode:
    def __init__(self, key, val, left=None, right=None, parent=None):
        self.key = key              # 用来比较和查找的 key
        self.payload = val          # key 对应的数据 data / value
        self.leftChild = left       # 左子树引用
        self.rightChild = right     # 右子树引用
        self.parent = parent        # 父节点引用，删除和旋转时很有用

    def hasLeftChild(self):
        return self.leftChild

    def hasRightChild(self):
        return self.rightChild

    def isLeaf(self):
        return not (self.leftChild or self.rightChild)

    def hasAnyChildren(self):
        return self.leftChild or self.rightChild

    def hasBothChildren(self):
        return self.leftChild and self.rightChild
理解：BST 是“链接式”的树，不像列表靠下标找元素。每个节点都保存左右子节点引用，整个树从 root 一路连下去。

3. BinarySearchTree 类：整棵树要记住什么

代码解释：BST 对象本身只需要 root 和 size
class BinarySearchTree:
    def __init__(self):
        self.root = None
        self.size = 0

    def length(self):
        return self.size

    def __len__(self):
        return self.size

四、BST 的 put/get 与迭代器

1. put：插入 key-val

put 的思想：空树直接作为根；非空树从 root 开始比较，key 小就去左子树，key 大就去右子树，直到找到空位置插入。
代码解释：put 负责入口，_put 负责递归向下找位置
def put(self, key, val):
    if self.root:
        self._put(key, val, self.root)
    else:
        self.root = TreeNode(key, val)
    self.size += 1


def _put(self, key, val, currentNode):
    if key < currentNode.key:
        if currentNode.hasLeftChild():
            self._put(key, val, currentNode.leftChild)
        else:
            currentNode.leftChild = TreeNode(key, val, parent=currentNode)
    else:
        if currentNode.hasRightChild():
            self._put(key, val, currentNode.rightChild)
        else:
            currentNode.rightChild = TreeNode(key, val, parent=currentNode)
代码题提醒：真正写 Map 时，若 key 已存在，通常应更新 payload，而不是再插入重复 key。课件代码主要突出 BST 插入路径。

2. 让 BST 像字典一样赋值

特殊方法：支持 mytree[3] = "red"
def __setitem__(self, k, v):
    self.put(k, v)

3. get：根据 key 找 payload

代码解释：找到节点就返回 payload；找不到返回 None
def get(self, key):
    if self.root:
        res = self._get(key, self.root)
        if res:
            return res.payload
    return None


def _get(self, key, currentNode):
    if not currentNode:
        return None
    elif currentNode.key == key:
        return currentNode
    elif key < currentNode.key:
        return self._get(key, currentNode.leftChild)
    else:
        return self._get(key, currentNode.rightChild)
比较当前节点→相等返回/小走左/大走右

4. __getitem__、__contains__ 与 __iter__

像字典一样读值、判断归属
def __getitem__(self, key):
    return self.get(key)


def __contains__(self, key):
    return True if self._get(key, self.root) else False
TreeNode 的迭代器：中序遍历，yield 逐个返回 key
def __iter__(self):
    if self:
        if self.hasLeftChild():
            for elem in self.leftChild:
                yield elem
        yield self.key
        if self.hasRightChild():
            for elem in self.rightChild:
                yield elem
理解 yield：它不是一次性返回整个列表，而是每访问到一个 key 就“吐出”一个结果。因为这里按中序遍历，所以对 BST 迭代会按从小到大的顺序输出 key。

五、BST 删除：三种情况

1. delete 的整体流程

delete：先用 _get 找到要删的节点；找到了调用 remove；找不到就报错。真正麻烦的是 remove，因为删完以后还必须保持 BST 的左小右大。
代码解释：delete 是入口，remove 才是核心
def delete(self, key):
    if self.size > 1:
        nodeToRemove = self._get(key, self.root)
        if nodeToRemove:
            self.remove(nodeToRemove)
            self.size -= 1
        else:
            raise KeyError('Error, key not in tree')
    elif self.size == 1 and self.root.key == key:
        self.root = None
        self.size -= 1
    else:
        raise KeyError('Error, key not in tree')


def __delitem__(self, key):
    self.delete(key)

2. 情况一：被删节点是叶节点

处理：最简单，直接让父节点对应的 leftChild 或 rightChild 指向 None。

3. 情况二：被删节点只有一个子节点

处理：让唯一子节点上移，接到被删节点原来的父节点下面。如果被删节点本身是根，就用子节点的数据替换根。

4. 情况三：被删节点有两个子节点

处理：找“后继节点”替换它。后继节点就是被删节点右子树中最小的节点，也就是右子树一路向左走到底。
找到被删节点→找右子树最小节点→摘出后继→用后继 key/payload 替换
为什么后继最多只有一个子节点？因为后继是右子树中最小的节点，它不可能还有左子节点；否则左子节点会更小，和“最小”矛盾。
考试够用的 remove 思路骨架
def remove(self, currentNode):
    if currentNode.isLeaf():
        # 情况1：叶节点，断开父节点指针
        ...
    elif currentNode.hasBothChildren():
        # 情况3：两个子节点，用后继替换
        succ = currentNode.findSuccessor()
        succ.spliceOut()
        currentNode.key = succ.key
        currentNode.payload = succ.payload
    else:
        # 情况2：只有一个子节点，让孩子顶上来
        ...

例题2：删除 BST 节点时为什么不能随便拿一个孩子顶上来？

答案：如果被删节点有两个子节点，随便拿左孩子或右孩子上移可能破坏 BST 的大小关系。用“右子树中最小的后继”替换，可以保证它大于左子树所有节点，又小于右子树其他节点。

六、BST 复杂度与退化

1. BST 操作的成本看树高

核心：put/get/in/del 的路径长度，都和树的高度 h 有关。树越矮，找得越快；树越歪，找得越慢。
BST 形状高度put/get 数量级原因
比较平衡约 log₂nO(log n)每层大约排除一半节点
退化成链约 nO(n)每次只少一个节点，像顺序查找

2. 为什么需要 AVL

普通 BST 的问题不是“思想错”，而是它不能保证平衡。只要树变得很歪，O(log n) 的优势就没了。AVL 树就是为了解决这个问题：插入后发现不平衡，就通过旋转修正。

七、AVL 树与平衡因子

1. AVL 树是什么

AVL 树：一种会自动保持平衡的二叉查找树。它仍然满足 BST 的左小右大，但额外要求每个节点的左右子树高度差不能太大。

2. 平衡因子 balance factor

公式：balanceFactor = height(leftSubTree) - height(rightSubTree)

  • 平衡因子 > 0：左重。
  • 平衡因子 < 0：右重。
  • 平衡因子 = 0：左右高度相同。
  • AVL 要求每个节点的平衡因子只能是 -1, 0, 1。
平衡因子含义是否需要调整
0左右一样高不需要
1左边高 1 层仍允许
-1右边高 1 层仍允许
2 或 -2左右高度差过大需要旋转重新平衡

3. AVL 的性能为什么稳定

AVL 的最差形状也不能太歪。课件用斐波那契数列分析说明：节点数 N 和树高 h 之间仍保持对数关系，所以搜索时间复杂度是 O(log n)。

八、AVL 插入、更新与旋转

1. 插入新节点后，平衡因子怎么变

新 key 先按 BST 规则插入为叶节点。叶节点本身平衡因子为 0，但它会影响祖先节点：插入到父节点左边，父节点平衡因子 +1；插入到父节点右边，父节点平衡因子 -1。
BST 方式插入叶节点→更新父节点平衡因子→继续向祖先传递→必要时旋转
什么时候停止向上更新？到根节点为止，或者某个父节点平衡因子变成 0 时停止。因为从 ±1 变成 0 表示这棵子树高度没有继续增加，不会再影响更上层。

2. 旋转的四种情况

失衡类型看起来像处理方法一句话
LL 左左型左子树的左边太高右旋左边太重，把左孩子提上来
RR 右右型右子树的右边太高左旋右边太重，把右孩子提上来
LR 左右型左子树的右边太高先左旋左子节点，再右旋当前节点先变成 LL，再处理
RL 右左型右子树的左边太高先右旋右子节点，再左旋当前节点先变成 RR，再处理

3. 左旋转怎么理解

左旋口诀：右孩子 B 提上来，旧根 A 变成 B 的左孩子；B 原来的左子树 C 挪到 A 的右边。这样仍然保持 A < C < B < E 的 BST 顺序。

4. 复杂度

操作成本原因
查找 getO(log n)AVL 保证树高是 O(log n)
插入 putO(log n)插入路径和更新祖先最多 O(log n)
旋转O(1)最多调整有限几个指针，与 n 无关

十、OJ 例题应用

1. A01：给定树结构，填成 BST

例题3：A01 二叉查找树

给定一个二叉树结构和一组整数，把整数填入对应节点，使其成为 BST；输出填好后的层次遍历。

样例输出：58 25 82 11 38 67 45 73 42

核心套路：BST 的中序遍历是升序。因此先中序遍历“节点编号”，再把给定数字排序，按中序顺序填回节点，最后 BFS 层次遍历输出。
读入树结构→中序得到节点顺序→数字排序→按中序填值→层次遍历输出
代码骨架：中序填值 + BFS 输出
def inorder(idx):
    if idx == -1:
        return
    inorder(left[idx])
    order.append(idx)       # 记录中序访问到的节点编号
    inorder(right[idx])

values.sort()
inorder(root)

for node_idx, val in zip(order, values):
    ans[node_idx] = val      # 中序位置填升序值

# 层次遍历输出
from collections import deque
q = deque([root])
res = []
while q:
    x = q.popleft()
    res.append(ans[x])
    if left[x] != -1:
        q.append(left[x])
    if right[x] != -1:
        q.append(right[x])
print(*res)

2. A02：完全二叉查找树

例题4：A02 完全二叉查找树

给定一系列整数，构造既是 BST 又是完全二叉树的树，输出层次遍历。

样例输入：1 2 3 4 5 6 7 8 9 0

样例输出：6 3 8 1 5 7 9 0 2 4

关键理解：完全二叉树的结构由 N 决定，可以用数组表示；BST 的中序是升序。所以只要对“完全二叉树数组下标”做中序遍历，并按排序后的值依次填入即可。
代码骨架：数组表示完全二叉树，0 下标时左=2i+1，右=2i+2
def inorder_index(i):
    if i >= n:
        return
    inorder_index(2*i + 1)       # 左孩子
    result[i] = nums.pop(0)      # 中序位置填入当前最小值
    inorder_index(2*i + 2)       # 右孩子

nums = sorted(nums)
n = len(nums)
result = [None] * n
inorder_index(0)
print(*result)                   # 数组本身就是层次遍历顺序
优化提醒：pop(0) 在 Python list 中是 O(n)，数据量大时可以用下标指针代替。

3. A03：从 BST 到更大和树

例题5：A03 从二叉搜索树到更大和树

给定一个 BST，把每个节点的值改成原树中所有大于等于它的节点值之和；输出修改后的层次遍历。

样例输入：6 1 8 3 4 9 2 7 5 0

样例输出：30 45 17 45 42 24 9 44 39 35

核心套路：BST 中序是升序，反向中序“右 → 根 → 左”就是降序。按降序走，用一个累积和 s 记录已经见过的更大值。
代码骨架：反向中序遍历维护累积和
total = 0

def convert(node):
    global total
    if not node:
        return
    convert(node.right)      # 先处理更大的
    total += node.val        # 加上当前原值
    node.val = total         # 改成大于等于它的和
    convert(node.left)       # 再处理更小的

4. POJ242：只给前序和后序，数 M 叉树可能数量

例题6：POJ242 Pre-Post-erous!

给定 M 叉树的前序遍历和后序遍历，一般不能唯一确定树，要计算可能产生多少种不同的树。

样例：2 abc cba 的输出是 4。

思路：前序第一个、后序最后一个都是根。去掉根后，把剩下的前序/后序同步切成若干子树；每棵子树递归计算可能数量。若当前根实际有 k 棵子树，而 M 叉树有 M 个槽位，就还要乘上组合数 C(M,k)。
