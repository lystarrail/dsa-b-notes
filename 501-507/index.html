一、本章考什么、怎么学

本章主线：查找是在数据里找目标；排序是先把数据整理成有序状态。排序本身也常常是为了让后续查找更快，比如先排序再多次二分查找。
顺序查找O(n)
二分查找O(log n)
简单排序多为 O(n²)
分治排序常见 O(n log n)

1. 本章知识地图

2. 常见考点

类型重点做题提醒
查找分析顺序查找有序/无序的最好、最坏、平均比对次数；二分查找 O(log n)。二分查找必须建立在“已排序”的前提上。
排序过程冒泡、选择、插入、快速的“一趟/一次划分”结果。手算时不要只看最终排序，要看每一趟的中间状态。
复杂度O(n)、O(log n)、O(n²)、O(n log n)。快速排序平均 O(n log n)，最坏 O(n²)。
稳定性与空间冒泡、插入、归并通常稳定；选择、希尔、快速通常不稳定；归并需要额外空间。“稳定”只讨论相等关键字的相对顺序。

二、顺序查找：从头到尾逐个比对

1. 什么是顺序查找

顺序查找：从列表第一个元素开始，按照下标增长的顺序，一个一个与目标 item 比较；找到就成功，走到最后还没找到就失败。

2. 无序表顺序查找代码

代码解释：无序表 sequentialSearch
def sequentialSearch(alist, item):
    pos = 0
    found = False

    while pos < len(alist) and not found:
        if alist[pos] == item:
            found = True          # 找到了，循环可以停
        else:
            pos = pos + 1         # 没找到，看下一个位置

    return found

testlist = [1, 2, 32, 8, 17, 19, 42, 13, 0]
print(sequentialSearch(testlist, 3))    # False
print(sequentialSearch(testlist, 13))   # True

3. 复杂度：为什么是 O(n)

情况最好最坏平均
目标在列表中第 1 个就是，1 次最后一个才是，n 次大约 n/2 次
目标不在列表中n 次n 次n 次
理解：无序表没有办法提前判断“后面一定没有”。所以没找到时必须一路查到最后。

4. 有序表顺序查找：可以提前停，但仍是 O(n)

代码解释：有序表 orderedSequentialSearch
def orderedSequentialSearch(alist, item):
    pos = 0
    found = False
    stop = False

    while pos < len(alist) and not found and not stop:
        if alist[pos] == item:
            found = True
        else:
            if alist[pos] > item:
                stop = True       # 当前值已超过目标，后面更不可能有
            else:
                pos = pos + 1

    return found

例题1：顺序查找平均比较次数

题目思想：长度为 3 的顺序表，查找第 1、2、3 个元素的概率分别为 1/2、1/4、1/8，另有 1/8 的概率查找失败，平均需要比较多少次？

参考答案：1×1/2 + 2×1/4 + 3×1/8 + 3×1/8 = 7/4。失败时要比较 3 次。

三、二分查找：每次把范围缩小一半

1. 二分查找的前提

二分查找必须要求列表有序。没有顺序，就不知道中间项比目标大/小之后该往左找还是往右找。
每次比较都能排除大约一半候选项，所以数量级从 O(n) 降为 O(log n)。

2. 迭代版二分查找代码

代码解释：不用切片，使用 first / last 索引
def binarySearch(alist, item):
    first = 0
    last = len(alist) - 1
    found = False

    while first <= last and not found:
        midpoint = (first + last) // 2
        if alist[midpoint] == item:
            found = True
        else:
            if item < alist[midpoint]:
                last = midpoint - 1    # 目标只可能在左半边
            else:
                first = midpoint + 1   # 目标只可能在右半边

    return found

3. 递归版二分查找与切片问题

代码解释：递归版更像“分治”，但切片有额外开销
def binarySearchRec(alist, item):
    if len(alist) == 0:
        return False
    midpoint = len(alist) // 2
    if alist[midpoint] == item:
        return True
    elif item < alist[midpoint]:
        return binarySearchRec(alist[:midpoint], item)      # 切片复制左半边
    else:
        return binarySearchRec(alist[midpoint+1:], item)    # 切片复制右半边
易错点：理论上二分查找比较次数是 O(log n)，但如果递归代码频繁使用切片，切片本身要复制列表，会带来额外时间和空间开销。实际写高效代码时更推荐传索引边界。

例题2：二分查找一定比顺序查找划算吗？

题目思想：二分查找 O(log n)，顺序查找 O(n)，是不是任何情况下都应该先排序再二分查找？

参考答案：不一定。如果数据本来有序，或者一次排序后要查很多次，二分查找很划算；但如果数据经常变化、只查一两次，排序成本可能比直接顺序查找更不经济。

四、冒泡排序：相邻比较，逆序就交换

1. 冒泡排序在做什么

冒泡排序：每一趟从左到右比较相邻元素，若前一个大于后一个就交换。每趟结束后，当前未排序部分的最大值会“冒”到最后。

2. 冒泡排序代码

代码解释：passnum 表示本趟最后要比较到哪里
def bubbleSort(alist):
    for passnum in range(len(alist)-1, 0, -1):
        for i in range(passnum):
            if alist[i] > alist[i+1]:
                alist[i], alist[i+1] = alist[i+1], alist[i]

alist = [54, 26, 93, 17, 77, 31, 44, 55, 20]
bubbleSort(alist)
print(alist)

3. 改进：如果某一趟没有交换，就提前结束

代码解释：短冒泡 shortBubbleSort
def shortBubbleSort(alist):
    exchanges = True
    passnum = len(alist) - 1
    while passnum > 0 and exchanges:
        exchanges = False
        for i in range(passnum):
            if alist[i] > alist[i+1]:
                exchanges = True
                alist[i], alist[i+1] = alist[i+1], alist[i]
        passnum = passnum - 1
复杂度：普通冒泡比较次数是 (n-1)+(n-2)+...+1，所以是 O(n²)。改进版在列表已经有序或接近有序时可能很快，但最坏仍是 O(n²)。

五、选择排序：每一趟选出最大项，只交换一次

1. 选择排序比冒泡改进在哪里

选择排序：每一趟扫描未排序部分，找到最大值的位置，最后把它和本趟末尾交换。它减少的是交换次数，不减少比较次数。
算法每趟怎么做比较次数交换次数
冒泡排序相邻逆序就交换，最大项一路冒到最后O(n²)最坏 O(n²)
选择排序先找最大项位置，最后才交换一次O(n²)O(n)

2. 选择排序代码

代码解释：positionOfMax 记录本趟最大值的位置
def selectionSort(alist):
    for fillslot in range(len(alist)-1, 0, -1):
        positionOfMax = 0
        for location in range(1, fillslot+1):
            if alist[location] > alist[positionOfMax]:
                positionOfMax = location

        alist[fillslot], alist[positionOfMax] = alist[positionOfMax], alist[fillslot]
稳定性提醒：直接选择排序通常不稳定。因为它会把最大/最小元素和某个远处位置交换，可能改变相等关键字的原有相对顺序。

六、插入排序：维护一个已排序子表

1. 插入排序像整理扑克牌

插入排序：假设列表前部已经排好序，每次拿一个“新项”，从右往左比较，把比它大的元素右移，最后把新项插入空位。

2. 插入排序代码

代码解释：currentvalue 是“新项”，position 向左移动
def insertionSort(alist):
    for index in range(1, len(alist)):
        currentvalue = alist[index]
        position = index

        while position > 0 and alist[position-1] > currentvalue:
            alist[position] = alist[position-1]   # 右移较大的元素
            position = position - 1

        alist[position] = currentvalue            # 插入新项

3. 复杂度与特点

  • 最坏情况：逆序，每个新项都要一路移到前面，O(n²)。
  • 最好情况：已经有序，每趟只比较一次，O(n)。
  • 移动操作只是一条赋值，通常比冒泡中的交换更省。

例题3：插入排序第四趟结果

题目思想：关键字序列 (46, 74, 16, 53, 14, 26, 40, 38, 86, 65, 27, 34)，用直接插入排序从小到大排序，写出第四趟后的排列。

参考过程：第 1 趟插入 74；第 2 趟插入 16；第 3 趟插入 53；第 4 趟插入 14。

参考答案：14, 16, 46, 53, 74, 26, 40, 38, 86, 65, 27, 34。

七、谢尔排序：先分组插入，再整体插入

1. 为什么要有谢尔排序

谢尔排序：以插入排序为基础，先按较大间隔把列表分成若干子列表，对每个子列表做插入排序；再逐步缩小间隔，最后间隔为 1 时做普通插入排序。

2. 谢尔排序代码

代码解释：gap 逐步变小，gap=1 时就是普通插入排序
def gapInsertionSort(alist, start, gap):
    for i in range(start + gap, len(alist), gap):
        currentvalue = alist[i]
        position = i
        while position >= gap and alist[position-gap] > currentvalue:
            alist[position] = alist[position-gap]
            position = position - gap
        alist[position] = currentvalue


def shellSort(alist):
    sublistcount = len(alist) // 2
    while sublistcount > 0:
        for startposition in range(sublistcount):
            gapInsertionSort(alist, startposition, sublistcount)
        sublistcount = sublistcount // 2
复杂度：谢尔排序的严格分析较复杂，通常介于 O(n) 和 O(n²) 之间，取决于间隔序列。它的核心价值是：先把表变得接近有序，再让插入排序发挥优势。

八、归并排序：分裂成小表，再有序合并

1. 归并排序的递归三步

归并排序：分裂、排序、归并。先把表不断分成两半，直到每个小表只有 1 个元素；再把两个有序小表合并成更大的有序表。

2. 归并排序代码

代码解释：左右两半各自排好，再“拉链式”合并
def mergeSort(alist):
    if len(alist) > 1:
        mid = len(alist) // 2
        lefthalf = alist[:mid]
        righthalf = alist[mid:]

        mergeSort(lefthalf)
        mergeSort(righthalf)

        i = j = k = 0
        while i < len(lefthalf) and j < len(righthalf):
            if lefthalf[i] <= righthalf[j]:
                alist[k] = lefthalf[i]
                i += 1
            else:
                alist[k] = righthalf[j]
                j += 1
            k += 1

        while i < len(lefthalf):
            alist[k] = lefthalf[i]
            i += 1
            k += 1

        while j < len(righthalf):
            alist[k] = righthalf[j]
            j += 1
            k += 1

3. 复杂度与空间

  • 分裂层数约为 log n。
  • 每一层归并时，所有元素都会被处理一次，约 O(n)。
  • 总时间复杂度：O(n log n)。
  • 需要额外数组空间，空间复杂度约 O(n)。
易错点：归并排序通常稳定，但前提是相等时优先取左半边元素，例如代码里使用 <=。

九、快速排序：选基准、分区、递归

1. 快速排序在做什么

快速排序：选一个基准值 pivot，把比它小的放左边，比它大的放右边，让 pivot 到达最终位置；然后递归排序左右两边。

2. 快速排序代码框架

代码解释：quickSortHelper 递归处理左右区间
def quickSort(alist):
    quickSortHelper(alist, 0, len(alist)-1)


def quickSortHelper(alist, first, last):
    if first < last:
        splitpoint = partition(alist, first, last)
        quickSortHelper(alist, first, splitpoint-1)
        quickSortHelper(alist, splitpoint+1, last)
代码解释：partition 找到基准值最终位置
def partition(alist, first, last):
    pivotvalue = alist[first]
    leftmark = first + 1
    rightmark = last
    done = False

    while not done:
        while leftmark <= rightmark and alist[leftmark] <= pivotvalue:
            leftmark += 1
        while rightmark >= leftmark and alist[rightmark] >= pivotvalue:
            rightmark -= 1

        if rightmark < leftmark:
            done = True
        else:
            alist[leftmark], alist[rightmark] = alist[rightmark], alist[leftmark]

    alist[first], alist[rightmark] = alist[rightmark], alist[first]
    return rightmark

3. 快速排序复杂度

情况原因复杂度
平均/较好每次分区比较均衡，递归深度约 log n，每层扫描 n 个元素O(n log n)
最坏每次 pivot 都把表分成一边空、一边 n-1，递归深度接近 nO(n²)

例题4：快速排序第一次划分

题目思想：关键码 [46,70,56,38,40,80]，利用快速排序，以第一个记录 46 为基准，第一次划分后可以得到什么结果？

参考答案：一种合理结果是 [38,40,46,56,70,80]。这类题重点是：46 已经被放在分裂点，左边都小于 46，右边都大于 46；不同实现细节可能导致左右内部顺序略有差异。

例题5：哪个排序算法是稳定的？

题目思想：以下典型排序算法中，具有稳定排序特性的是：A. 冒泡排序 B. 直接选择排序 C. 快速排序 D. 希尔排序。

参考答案：A. 冒泡排序。冒泡排序只在相邻元素逆序时交换；相等元素通常不会交换位置，所以稳定。选择、快速、希尔通常不稳定。

例题6：最坏情况复杂度相同

题目思想：哪一组中的两个排序算法最坏情况下时间复杂度的大 O 表示相同？

选项：A. 快速排序和选择排序　B. 归并排序和快速排序　C. 插入排序和归并排序　D. 冒泡排序和归并排序

参考答案：A. 快速排序和选择排序。快速排序最坏可退化为 O(n²)，选择排序最坏也是 O(n²)。

考试提醒：“最快”不是唯一标准。归并稳定但需要额外空间；快速平均很快但最坏 O(n²)；插入排序在接近有序时很强；冒泡排序可通过是否发生交换提前结束。
